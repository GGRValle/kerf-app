F-EST1 · Spec

The only divergence — lead capture

  • Who's this for? — find an existing client OR add a new lead inline: Name · Phone · Email · Address (geo-locate).
  • Project # is Assigned on save — there's no project yet; saving the estimate creates the opportunity record.
  • Coral tint signals "new lead / sales lane" without changing the layout — it's still the same builder.

Shared with F-CHG1 (verbatim)

  • RH-assisted banner → blurred one-surface voice overlay.
  • Title · Type (Lump sum / Sections / Itemized layout) · Scope · Add item → same template categories (Assemblies · Items · Materials · Labor · Subcontractor · Demolition).
  • Totals auto-calc by permission + settings; integer-cents storage; nothing posts without review.

Send is the pre-contract gate

  • A signed estimate becomes a contract (per the pre-contract canon, F-PV1). Send routes to send-for-signature, an explicit external gate — never autonomous.
  • On signature, the lead promotes to a client + project and the next change order can prefill from it (→ F-CHG1).
