C · Kerf-as-MCP-server — system-of-record positioning V2.0α direction · no V1 schema impact · moat extender via integration surface
KERF · MCP SERVER · CLIENTS
Kerf exposes its Blackboard and projections as a typed MCP interface. Other AI tools — Claude Code, Cowork, ChatGPT desktop, IDEs, custom agents — become clients of Kerf, not competitors. The contractor's full AI stack reads canonical state from Kerf. Permission lattice + authority lattice + privilege bypass enforce role-scoped access at the MCP boundary; clients inherit the same controls automatically.

Why this is a moat extender: the more agents that talk to Kerf, the stickier Kerf gets. Switching cost rises with every integration. Kerf becomes the system of record in the contractor's AI stack — other tools handle their own thing (drafting, search, reasoning) but the canonical state lives in Kerf. Houzz Pro and Buildertrend cannot honestly claim this because their data models aren't typed for external consumption.
D · Automation guardrails — token + spend + class envelope tenant settings · V1 schema · automation autonomy bounded by three layers
AUTOMATION · GUARDRAIL ENVELOPE
Tenants get autonomy to set up automations and let Right Hand act on their behalf. Three guardrails ensure that autonomy does not become liability. All trips logged immutably. All halts surface to the owner.

$0 $1,402 · 78% Cap: $1,800 · Team Pro
Soft ceiling at 80% ($1,440) → notify owner, recommend tier upgrade. Hard ceiling at 100% ($1,800) → essential workflows continue, non-essential automations halt with metered opt-in.

0 avg 24K · max 47K Cap: 50K input + 8K output
Single agent invocation cap: 50K input + 8K output tokens. Single chain cap: 5 invocations before checkpoint. Both enforced at the LLM gateway. Trips logged.
