Kerf · Views Master · v1.0

Make more. Decisions, not dashboards.

Single canonical wireframe package. Voice-driven decision making — every operator interaction starts with the persistent voice button. Tap = capture session (intent declared by first words; system surfaces camera / LiDAR / dimensions per intent). Hold = Right Hand conversation mode. Multimodal evidence (voice + photo + LiDAR + GPS + actor + project) bundles into one EvidenceObject; Right Hand observes, retrieves from Cost KB / Blackboard / prior decisions, drafts a Decision Packet, validators check it, operator approves / edits / rejects in one tap.

The metaphor: Right Hand is the assistant who runs to the library, types up what you need, runs it back, and double-checks — all while you stay in the work. Source-or-silent. Mobile-first. Spanish-native. Decisions, not dashboards.

The model — three states, one button

  • Tap (state 1 → 2) — enters capture session. First 3-5s of voice classifies intent: estimate · change order · daily log · site visit · punch list · Voice Tour · approval-gate authoring · or general capture.
  • Intent → contextual tools (state 2). Per intent, the canvas surfaces:
    • Estimate / new project — voice + camera + LiDAR (room scan)
    • Change order field — voice + camera + dimension/measure
    • Daily log — voice + camera
    • Site visit — voice + camera + LiDAR (targeted)
    • Voice Tour — voice only (structured dialog)
    • Approval-gate authoring — voice + camera (for receipts/docs)
  • All modalities timestamp into one EvidenceObject with project, captured_at, captured_by, GPS, actor, intent. Voice keeps recording while operator snaps a photo or starts a LiDAR scan.
  • Hold (state 3) — Right Hand conversation. Two-way dialog. RH retrieves, drafts, surfaces. Voice-driven decision making.
  • Recording state — button turns alert-red with stop affordance per canon mobile v2 line 1512.

The "running to the library" model

  • Operator stays in the work. Voice button captures the messy reality of the jobsite (narrative + pictures + dimensions).
  • Right Hand observes the EvidenceObject → retrieves from Cost KB, prior decisions, project state, sub relationships, Blackboard projections.
  • Right Hand drafts a Decision Packet with sourced lines (per v0.2 §5.1 schema · source-or-silent).
  • Validator wall (V1, V2, V6, V7, V8, V9, V12, V17, V18) checks the draft.
  • Decision Packet surfaces in operator's queue + notification routes via configured channel.
  • Operator approves / edits / rejects in one tap.
  • Approved decision generates artifacts (CO PDF, e-sign link, schedule update, etc.). Memory effects feed LearningSignals. Compounds.

What this enables (canon citations)

  • Pathway 1 evidence ingestion (Research Report v0.2 §6.2) — daily field captures.
  • Pathway 4 Voice Tour onboarding (§6.5) — same UI affordance, structured-dialog intent.
  • Approval-gate authoring (intake_workflow.md §J) — voice as primary input to revise prices + lines at every threshold.
  • Spanish-native field crew interaction (project_kerf_spanish_phase_decision.md) — voice with auto-detect EN/ES from V1.
  • D-024 Demo Spine — "field signal capture (voice + photo)" is the load-bearing capture primitive.

Three equal paths to the same outcome

  • ◉ Voice (primary) — tap to capture, hold to converse. Most operations use this.
  • 💬 Chat (alternative) — typed input. Same intent declaration, same RH conversation, same EvidenceObject output. Used for: loud jobsites, accessibility, offline (with queueing), Zoom calls, library/quiet environments, ESL situations where typing is more accurate.
  • ☰ Tools (shortcut) — slide-up sheet of common intents. Skip voice entirely: tap "Camera" → enter capture session with camera primed. Tap "Change order" → CO intake form opens. Tap "New intake" → project intake mode picker.

Why each one matters

  • Voice is fastest in 80% of contexts. But the system is unusable if it's the only option.
  • Chat path enables ADA-grade accessibility — speech impairment, hearing impairment with visual feedback only, situations requiring quiet.
  • Offline support: chat input queues; voice transcripts queue; both flush when connectivity returns. Critical for jobsites with poor signal.
  • Tools menu speeds up "I just need to take a photo" or "I'm starting a CO right now" without going through intent declaration.

What's stored regardless of input mode

  • Same EvidenceObject schema. capture_mode field carries: 'voice' · 'text' · 'photo_only' · 'lidar_only' · 'multimodal'.
  • Same source-layer chips — input mode doesn't affect provenance.
  • Same Decision Packet routing — Right Hand observes regardless of how the EvidenceObject was captured.

Std/Graph toggle moved

  • The Std/Graph view toggle (canon mobile v2 §operator) was previously on the bottom bar. It's now inside the ☰ tools sheet under "View" — less central, since the toggle is a per-session preference rather than a primary action.

What RH does in conversation mode

  • Listens — bilingual auto-detect (EN/ES). Mano Derecha is the canonical ES name.
  • Retrieves — Cost KB · Blackboard projections · prior decisions · project state · sub relationships. Source-or-silent.
  • Acts — approve / edit / reject Decision Packets via voice; send messages; update schedules; query state; undo.
  • Cites — every claim in the response carries source-layer chips inline.
  • Audits — voice transcript becomes the source for every action triggered. Blackboard event with source = voice_session_id.
  • Confirms — Recap of what was done. Undo window 60s per canon undo state.

What RH never does

  • Send external comms without explicit voice approval.
  • Silently promote pricing / markup / assemblies to TENANT_MEMORY.
  • Invent costs, dates, or scope. If TENANT_MEMORY has no answer, RH asks.
  • Refuse a price override. Owner always wins on price; override is logged.
  • Treat urgency as authority. Authority lattice routes by role, not by exclamation.
