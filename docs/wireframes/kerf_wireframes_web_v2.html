Intake workflow — Project + Change Order (V1.5-V2.0)One architecture, two entry points. Multi-modal: form / voice / recorded-conversation-with-consent / text. Photos + editable LiDAR (V2.0 Alpha). CO Intake reuses engine with project-context pre-populated and camera+LiDAR primed at top of form. Prominent intake CTA on Dashboard; CO button inside Project page (Frame 2 hosts CO Intake in V2.0).
FinTech partner-first (V2.0-V2.1)5 primitives, partner-first: homeowner pay + purchasing (V2.0), crew + job cards + financing referral (V2.1). Job-cards in-house is a V3.0+ decision only. Financing partner-forever (regulated lending). No fintech UI in V1 wireframes.
V1 scope unchangedThis addendum is architectural language only. The 8 frames below are the same as Apr 21 — no new screens, no new surfaces. Kill-switch criteria unchanged.

Why 60 seconds. The fear Christian named in the context brief: "a rogue agent sending a bad message to a major client is worse than no AI at all." The 60s window costs nothing operationally, removes the panic of a bad click, and demonstrates control. Under the hood: Policy Engine (v3.2 §12) holds the send for 60s, any undo during that window rolls the Locked state back to Approved-pending with a one-line note in the audit trail.
Applied to: external SMS/email sends · change order dispatch · QuickBooks writes (invoices, payments) · policy changes in Kerf Teams. Not applied to: internal reads, internal drafts, agent-internal work. 60s is configurable per tenant in Kerf Teams.

WHY THIS SURFACE EXISTS
Architecture v3.2 §9.2 is explicit. Homeowners don't use the app. They don't log in, don't learn Standard or Graphical, don't see the queue. What they get is a phone-gated magic-link page the moment an approval is required, wrapped in GGR branding. Signature persists back to the same audit trail the operator side sees.
Two cards, one signature, one button. No navigation. The "why" is the first sentence. Photos are GPS + timestamp verified at capture (Source → Tag → Route chain). Cursive signature is auto-rendered from the registered signer's typed name — e-sig hash binds to device fingerprint + timestamp.
60-second undo lives on the operator side, not here. When Sarah approves, the Messenger agent doesn't immediately post to QuickBooks. Christian gets a 60s window to pull it back if something looks off. The homeowner's experience is still "done."
Not built yet — tier-three option. If a homeowner asks for a persistent portal (milestones, photo library, document history), that's a separate surface. Default is always SMS thread + magic-link. Portal is opt-in, never forced.
