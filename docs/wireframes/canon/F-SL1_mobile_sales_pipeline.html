F-SL1 · Spec

Locked Sales canon applied

  • Pipeline-first, proposal-enabled, relationship-aware, pre-deal only.
  • Pipeline IS the primary lens · Proposals is reached through deal detail, not surfaced as a peer here
  • Leads = stage inside Pipeline, not a separate sub-tier
  • Stage tags: Lead · Qualified · Bid · Sent · Negotiation (Won + Lost compress into footer / past view)

Sort logic

  • Section 1 · Needs attention — stale leads · overdue proposals · negotiations awaiting response · ready-to-send deals
  • Section 2 · On track — bids progressing on schedule · fresh leads with future qual scheduled
  • Within "needs attention," urgency-sorted: red (overdue) → amber (your action) → green (ready to send)

Mobile bar update

  • Owner default swaps revised: Home · Projects · Speak · Sales · More
  • Clients (was in default) drops to More sidebar (matches desktop demotion)
  • Sales takes the freed swap slot · matches Owner's lived daily attention
  • Other domains (Schedule · Money · Team+Ops · Marketing) reached via More

Row anatomy

  • Deal name · person or entity contracting
  • Stage tag · text-only label (Lead · Qual · Bid · Sent · Neg)
  • Meta line · scope summary + lifecycle context
  • Status chip (when applicable) · Stale · Overdue · Your response · Ready to send
  • State line · what's happening now
  • Amount · proposal $ or estimate · monospace

What this surface refuses to be

  • Not a CRM workstation with custom fields
  • Not a kanban board with drag-and-drop stage columns
  • Not Salesforce
  • Not a HubSpot pipeline view
  • Not a filter-and-sort dashboard
  • Not a separate "Proposals" peer view (proposals reached through deal detail)

Routes

  • Back arrow → F-A1 Owner Home
  • + New (top-right) → F-S1 Start sheet (estimate/proposal intent)
  • Deal row tap → F-SL3 Deal detail
  • + New estimate (dashed) → F-S1 Start sheet
  • Funnel footer tap → expanded pipeline analytics (V1.5+ surface · deferred)
  • 🎤 Speak → RH conversation pre-bound to Sales context
  • Bottom bar tabs → respective destinations · Sales stays active

Doctrine bound

  • One primary cognitive job — see pipeline state · drill where action lives
  • Smart summary leads — RH synthesis line · not stat grid
  • Sort by attention-needed · not alphabetical · not stage-grouped first
  • Stage as text tag · status colors reserved for state
  • Red is chip-tier · per feedback_red_is_chip_tier_not_row_tier.md
  • 3-color discipline · amber pending · green ready · red stale/overdue
  • Trust surfaces exception-tier · no transcript/audit nav
  • Right Hand named once · no agent names elsewhere

What's deferred

  • Forecast view (weighted pipeline) — V1.5+
  • Lost-deal analysis surface — V1.5+
  • Activity timeline aggregation across deals — V1.5+
  • Bulk actions on deals (mark multiple won/lost) — V2+
  • Stage drag-and-drop (avoid kanban temptation) — never

F-SL1 Grid · Spec

Big-picture canon applied

  • Comparison view · every deal visible at once
  • 2-column mobile · no attention compression
  • Tile content disciplined: name + stage + state chip + deal value/rhythm · scan not detail
  • Map view (V1.5+) would be third toggle option · deferred
What this surface enforces Pipeline state, not deal database. First sentence tells the operator what's happening across the funnel. List sorts attention-first so stale and overdue land at the top. Stage tags are quiet text labels; status carries the urgency. Pipeline funnel footer summarizes the funnel shape; doesn't dominate.
What this surface refuses to be Not Salesforce. Not HubSpot. Not a kanban board. Not a custom-field forest. F-SL1 is the pipeline synthesis with drill affordances underneath. Proposals don't get a peer view here — they live inside deal detail because a proposal exists in service of a deal.
