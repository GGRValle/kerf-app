F-AD2 · Spec

Build position

Tier 2 build · Team & Ops · Admin
✓F-AD1 · Mobile Admin landing
2F-AD2 · Desktop Admin landing (this)
3F-AD3+ · Admin item detail surfaces (per phase)

Desktop density choices

  • 4-column row at desktop width · name + phase | plain-English context | state + days-left | arrow. Mobile stacked all three into one block.
  • Right rail-led queue synthesis · mobile had RH summary above the list; desktop puts RH + time-sensitive callout + queue summary + delegate-batch action in the rail.
  • Time-sensitive callout in rail · 3 day-count rows in a red-tinted card · pulls the clock-running items into the rail's primary read.
  • Delegate batch action in rail · single-click "Send batch" to assign all 3 delegate-ready items to bookkeeper · scales beyond per-row delegation.
  • Mini-tabs in header band · filter scope stays visible above the list (above the fold at 1700px)

Grid doctrine applied

  • canonDomain list = comparison · admin IS comparison (peer admin tasks)
  • canonList + Grid both required · List default attention-first · Grid for macro scan
  • d-1One primary cognitive job — what admin needs my hand, what can be delegated
  • d-2Smart summary leads — RH card opens the rail
  • d-3Sort by attention — Time-sensitive → Needs your hand → Delegate-ready
  • d-4Phase as text tag · status colors reserved for state
  • d-5Red is chip-tier only — never full-row red border
  • d-6No agent names · "renewal due" not "Compliance Agent flagged"
  • d-7Translate canon vocab · "your admin work" not "admin queue cohort"
  • canonRight Hand named once · no other agent names in operator copy

Section order (locked)

  • Time-sensitive · <30 day clocks (license · COI · vehicle reg · permits with deadlines)
  • Needs your hand · approval / signature / review · owner-only authority
  • Delegate-ready · routine work safe to hand off · muted treatment
  • Cleared · Actions menu · not on default view

Mini-tabs (filter scope)

  • All · default · full admin queue (10 items)
  • Mine · only items where operator is action-holder (7)
  • Delegate-ready · items safe to hand to bookkeeper MCP (3)
  • Time-sensitive · items with <30d clock (3)

Fillable role

  • Same queue surfaces for Owner / external bookkeeper MCP / future Admin/Ops role-root
  • Role projection filters which items appear and what "Mine" means
  • Delegate-batch action targets the role (Bookkeeper), not a named person

Routes

  • New admin item (top-right) → New admin sheet (F-AD-NEW deferred)
  • Admin row click → F-AD3+ admin item detail (per-phase · deferred)
  • Time-sensitive row in rail → scroll list to that item
  • Delegate batch → send-to-bookkeeper sheet (F-AD-BATCH deferred)
  • Sidebar Team & Ops siblings (Crew · Subs · HR · Purchasing · Bookkeeping · Clients) → respective destinations
  • Sidebar Home → F-A2 · Owner Home (desktop)
  • Global search → cross-domain query

What this surface refuses to be

  • Not a bookkeeping ledger
  • Not a full document library
  • Not an HR roster (Crew is sibling sub-item)
  • Not a QuickBooks console
  • Not a compliance dashboard with charts and audit-trail widgets
  • Not a Houzz Pro / Buildertrend admin tab with 40 navigation items
  • Not a multi-tab admin hub

F-AD2 Grid · Spec

Macro-scan canon applied

  • Comparison view · every admin item visible at once
  • 4-column desktop grid · 16 tiles fit easily on one screen · shows the full annual admin landscape
  • Tile content disciplined: name + phase + meta + state chip + rhythm line
  • Delegate-ready tiles use neutral left border + dimmed opacity (no red/amber for routine work)
  • Grid extends past List by surfacing forward-looking annual items (SOS report · business licenses · ACA · 5500 · OSHA · handbook) without urgency chips — they're on the horizon, not in the queue this week
  • Drill into any tile → admin item detail (F-AD3+ deferred)

What Grid view refuses to be

  • Not a kanban (no drag between status columns)
  • Not a dashboard with micro-metrics per tile
  • Not a Houzz Pro admin grid with 40 fields per card
  • Scan-first · disciplined per the big-picture canon
What this surface proves at desktop density The mobile list works at desktop without becoming a transaction console. Adding columns, not features. The third column (plain-English context: who · what · why) gets its own zone instead of cramming into the name line. The admin rail carries the RH summary, the time-sensitive callout, the queue summary, and the delegate-batch action — so the main column stays a list, not a dashboard. CSLB renewal surfaces twice (once as a row, once as the top row of the time-sensitive rail card) — intentional: it's the highest-clock-pressure item.
What this surface refuses to be Not a bookkeeping ledger. Not a document library. Not a QuickBooks console. Not Buildertrend's admin sidebar with 40 navigation items. F-AD2 is F-AD1 with breathing room, plus the time-sensitive callout and delegate-batch action the rail can carry. A 1700px viewport spent on the same cognitive job — what admin needs my hand, what can be delegated — not on more widgets stitched together.
