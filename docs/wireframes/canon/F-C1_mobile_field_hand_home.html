F-C1 · Mobile Field Hand Home · Mano de Campo

The role-root proof point. Same 5-slot primitive as F-A1 Owner Home, but Field Hand projection — different defaults (Log + Clock), different content shape (today's work, not "one thing"), bilingual identity ("Habla" on the Speak button). Field Hand is not employee-mobile-mode; it's a teammate to the worker, scope-anchored. Today's work card shows what · where · how long · which scope line. Clock-in is the primary action. From RH section carries owner messages routed to crew. Per kerf_agent_naming.md and Operating Model Canon role-roots layer.

Sections (top to bottom)

  • Today's work — replaces Owner's "One Thing"; for crew, the work itself is the thing
  • My day — 4-cell strip: Hours · L-tier · Next job · Messages
  • Captures today — photo + voice notes recorded on site; flagged items surface to RH
  • From Right Hand — owner messages routed to crew; bilingual

Bar contract (Field Hand defaults)

  • MPos 1 · Home
  • FHPos 2 · Log (Field Hand default swap; replaces Owner's Projects)
  • MPos 3 · 🎤 Habla (mandatory · bilingual label)
  • FHPos 4 · Clock (Field Hand default swap; replaces Owner's Clients)
  • MPos 5 · More (sidebar opener)

Bilingual identity

  • BGreeting: Buenos días (locale-detected)
  • BSpeak button label: Habla (Mano de Campo bilingual cue per kerf_agent_naming.md)
  • BSection labels: dual English / Spanish (subtle, secondary)
  • BRH message preserved in Spanish if RH responded in Spanish
  • Tenant-level locale picks UI language; agent identity carries both languages as constant cue

Site context

  • Green "Hernandez site" pill in top chrome — GPS-matched to project address
  • Voice capture pre-binds to Hernandez automatically — crew doesn't say "for Hernandez"
  • If crew moves between sites in one day, pill updates and voice context follows
  • If no GPS match, pill omitted and voice prompts "which job?"

Today's work card

  • Headline: project · phase · day
  • Detail: hours expected · who's on it · address
  • Scope-line pill — green badge anchoring to work order > scope line
  • Primary CTA: Clock in (green, dominant)
  • Secondary: View scope (subdued)
  • Per Operating Model Canon doctrine #6: every field action anchors to scope and labor

Captures section

  • List of today's captures (photos + voice notes)
  • Per-item: thumbnail (photo glyph or voice glyph), title, meta, flag-for-RH indicator
  • Voice note shows duration + "transcript ready" when processed
  • Photos with EXIF location matching site auto-anchor to current Daily Log
  • "See all →" routes to Log destination (today's Daily Log)

From Right Hand

  • Direct messages from RH to this crew member
  • Often routed approvals: "Christian aprobó X · sigue adelante"
  • Bilingual: surfaces in the locale RH spoke in
  • Long-hold Speak FAB → opens RH conversation thread (V1.5+)

What this surface refuses to be

  • No surveillance dashboard (no "are you working?" framing)
  • No generic checklist detached from scope/labor
  • No Money / Sales / Admin / Client Success surfaces (those are Owner/PM concerns)
  • No agent name surfaced as a feature ("Drift agent flagged…")
  • No AltitudePacket / Policy Gate / cohort vocabulary

What changes vs F-A1 Owner Home

  • Green accent palette (Field) replaces amber accent (Owner) — visual role differentiation
  • Greeting + site context replace Owner's date-only chrome
  • "Today's work" replaces "One Thing"
  • "My day" 4-cell replaces "Business state" 4-cell — crew-scoped questions
  • "Captures today" + "From RH" replace "On Deck" + "The Pulse" — crew has different priorities
  • Bar swap defaults change: Projects + Clients → Log + Clock
  • Speak button label: "Speak" → "Habla" (when tenant locale = ES)
  • Avatar accent color reflects role (Field green vs Right Hand gold)

What stays identical

  • Same 5-slot bar primitive (3 mandatory + 2 swap)
  • Same monochrome silhouette icon system (More is the 4-color quadrant exception)
  • Same layered disclosure pattern
  • Same plain-English discipline
  • Same Right Hand voice (just routed crew-side here)
  • Same chrome dimensions and section labels

Routes (downstream)

  • Today's work card → Work order detail (scope-anchored execution surface)
  • Clock in CTA → time logged against scope line "framing (carpentry)"
  • Capture item · photo → Capture detail (EXIF + scope-anchor + Daily Log link)
  • Capture item · voice → Transcript view (per kerf_transcript_canon.md)
  • RH message → RH conversation thread
  • "My day" Hours cell → My Time history
  • "My day" L-tier cell → My Profile (L-ladder progress)
  • Log (Pos 2) → Daily Log destination (today + this week)
  • Clock (Pos 4) → Clock in/out destination
  • More (Pos 5) → Field Hand More sidebar (Schedule · other-crew · Messages · Profile · Settings)
