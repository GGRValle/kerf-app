F-SB2 · Spec

Build position

Tier 2 build · Team & Ops · Subs

Desktop density choices

  • 4-column row at desktop width · company + trade | contact + 1099 | state | arrow. Mobile stacked all three into one block.
  • Right rail-led roster synthesis · mobile had RH summary above the list and roster footer below; desktop puts both in the rail so the list dominates the main column.
  • COI watch in the rail · 4 day-grouped rows. Compliance signal that lives natively at desktop density (mobile relegated to footer summary).
  • SUB ids visible · meta line carries system id at desktop width without crowding (mobile dropped it).
  • Partner tenure visible · "3yr partner" surfaces relationship depth where mobile collapsed to jobs-YTD only.
  • Sidebar Team & Ops expanded · Subs active, siblings (Crew · HR · Purchasing · Bookkeeping · Clients) visible for context.

Doctrine bound (same as F-SB1, scoped to desktop)

  • d-1One primary cognitive job — see roster state, drill where risk lives
  • d-2Smart summary leads — rail RH card opens the page
  • d-3Sort by attention — no-show / liability → insurance expiring → bid quiet → calm → closed
  • d-4Trade as text tag · status colors reserved for state (red liability · amber compliance · cyan critical path · green stellar/cleared)
  • d-5Red is chip-tier, not row-tier · Westside Tile carries red chip, row stays visually neutral
  • d-6Roster summary in rail, not page header — reference content never leads
  • d-7No filter bar · default sort handles 80% · search via global chrome search
  • d-8Trust surfaces exception-tier — no audit or transcript nav on this page
  • canonRight Hand named once · no agent names elsewhere

Section order (locked)

  • Attention · subs with active flags (no-show · insurance · bid quiet)
  • Active subs · everyone in good standing, sorted by current rhythm
  • Closed / inactive · muted · re-engageable
  • Archive · long-press / Actions menu · not on default view

Routes

  • Add sub (top-right) → F-S1 · Start sheet → Add sub workflow
  • Sub row click → F-SBD · Sub Detail (jobs · COI · payments · threads)
  • Westside Tile at-risk callout → F-SBD · Westside Tile detail (drift state)

  • "All COIs" rail link → COI compliance overview (deferred)
  • Sidebar Crew → F-CR1 · Crew list (W2 sibling)
  • Sidebar Team & Ops parent → Team & Ops landing
  • Global search → cross-domain query

  • Not a Buildertrend subcontractor portal
  • Not a kanban with availability columns
  • Not a filter-and-sort dashboard with multi-select chips

F-SB2 Grid · Spec

Big-picture canon applied

  • Comparison view · every sub visible at once
  • 4-column desktop grid · 16 tiles fit easily on one screen · no attention compression
  • Tile content disciplined: company + trade + 1099 + contact + state chip + rhythm
  • State palette: red (no-show / liability) · amber (COI / bid quiet) · cyan (critical path / closing) · green (stellar)
  • Inactive subs muted (opacity) · stay on the grid for re-engagement scan
  • Map view (V1.5+) shown as third toggle option (deferred · geo-routed dispatch)
  • Drill into any tile → F-SBD Sub Detail

Why Grid matters for Subs at desktop

  • The whole bench fits in one viewport · 16 tiles at 4-wide is the real "show me everyone" view
  • Stellar-vs-flagged comparison is the core question on bid-out · grid puts both adjacent
  • Trade-by-trade visual scan: tile coverage, electrical coverage, plumbing coverage · roster shape is the answer
  • Inactive/closed subs muted but present · re-engagement is one click away

What Grid view refuses to be

  • Not a kanban (no drag between availability columns)
  • Not a dashboard with micro-metrics per tile
  • Not a Houzz Pro vendor card grid with 47 fields
  • Not a procurement portal
  • Scan-first · disciplined per the big-picture canon
What this surface proves at desktop density The mobile Subs list works at desktop without becoming a vendor database. Adding columns, not features. The third column (contact + 1099 status) gets its own zone instead of cramming into the name line. The roster rail carries the RH summary, at-risk callout, and COI watch — so the main column stays a list, not a dashboard. Westside Tile surfaces twice (once as a row, once as a rail callout) — that's intentional: it's the highest-attention item on the bench.
What this surface refuses to be Not a vendor database. Not a Buildertrend subcontractor portal. Not a procurement system with PO tracking. Not a tile dashboard with custom-column filters. F-SB2 is F-SB1 with breathing room, plus the COI watch the rail can carry. A 1700px viewport spent on the same cognitive job — bench state at a glance — not on more widgets stitched together.
