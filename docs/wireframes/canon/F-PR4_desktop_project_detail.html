v2 execution-first reframe (vs v1)

What changed (2026-05-18): First-load tilt was too scope-first in v1. Critique landed: "first page should feel closer to contract / state · schedule / who-what-where · invoicing / money posture." Reframe:

  • Lifted Field card + Schedule mini + Budget mini from the right rail into the main column as a 3-card execution row, before the tab strip
  • Dropped the 5-cell state strip (Phase/Moving/Blocked/Decision/Changed) — redundant with execution row content + RH sentence; Blocked "nothing right now" was filler
  • Slimmed the right rail to just CO-002 decision card + Audit deep-link
  • Synthesis row simplified from 2-column (RH + state strip) to single full-width RH sentence above the execution row
  • Result: ~32% of viewport above tabs reads as execution snapshot; tabs and scope content sit below as drill-downs

Desktop density choices (carried from v1)

  • Exclusions + Allowances side-by-side · two compact lists at half-width instead of stacked. Same density as mobile per list; desktop just fits two columns.
  • Actions in header · Log a touch · Send update · Actions menu in breadcrumb. Mobile had a single ⋯ action sheet; desktop affords the two most common actions inline.
  • PM name + Estimated complete in meta line · mobile collapsed to client + address; desktop fits PM and date.
  • Schedule mini scope is project-scoped · F-PR3 rail shows portfolio lookahead; this card shows just this project. Same data model, different scope by surface.
