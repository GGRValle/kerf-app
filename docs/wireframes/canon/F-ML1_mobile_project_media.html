4-question rule (F-FD1 pattern)

  • Macro / comparison / detail? Detail/browse — per-project media lens, not portfolio comparison.
  • List / grid / both? Thumbnail grid only · NO List/Grid toggle (detail surface · grid is primary).
  • Cognitive job? See what was captured on this job · filter to photos/videos/dailies · jump to daily log or fullscreen.
  • Hidden until drill? Full history (lazy-load) · fullscreen viewer · daily log body (F-E1 anchor).

Doctrine applied

  • d-1One primary cognitive job — browse project media, retrieve by voice or filter
  • d-2Smart summary leads — RH counts + last capture, not folder tree
  • d-3Red chip-tier only — damage flag on thumb overlay · no row border accent
  • d-4Plain contractor vocab — phase tags as text · no "extracted entities"
  • d-5Right Hand only — no agent names in body copy

Scale lens · 20–30 default · hundreds stress test

  • Default window: 24 thumbs visible (3-col mobile) · most recent sort
  • 248 total: lazy-load / infinite scroll · fetch next 24 on bottom threshold · skeleton placeholders during fetch
  • Stress test (500+): same grid shape · phase/date grouping chips compress scan · voice retrieval preferred for "show me framing week"
  • Performance note: thumbs are CDN-resized previews · full-res only in viewer

Capture integration · F-E1

  • Photo/video from F-E1 writes to Daily Log AND appears here · same file path + metadata in event log
  • Viewer action Open daily log → F-E1 daily log for capture date · scrolls to anchor
  • LiDAR / RoomPlan captures route to F-RC1 · still listed in Media with type badge (deferred badge in V1)

Voice retrieval routing

  • "Right Hand, take me to Wegrzyn pictures" → F-ML1 · project=Wegrzyn · filter=Photos · sort=Most recent
  • "Wegrzyn videos" → filter=Videos
  • "Most recent dailies from Hernandez" → project=Hernandez · filter=Daily logs
  • "Show me framing photos from last week" → filter=Photos · phase=Framing · date=last 7d
  • "Right Hand, label this framing day 3" → viewer open · voice caption edit (see Frame 3)

Routes

  • Back → F-PR2 Project Detail · Media context preserved
  • Thumb tap → fullscreen viewer (Frame 3)
  • Open daily log → F-E1 daily log anchor for that capture day
  • Share → OS share sheet · watermark optional (Settings)
  • 🎤 Speak → RH pre-bound to project + current filter

Viewer · Spec

Viewer anatomy

  • Full-bleed preview · pinch-zoom · swipe nav within filter set
  • Metadata: caption (editable via voice) · date · capturer · phase · project
  • Damage-flagged photos show red chip in metadata row · same chip-tier doctrine
  • Open daily log → F-E1 · date-scoped · scrolls to photo anchor in log body
  • Share → native share · optional Kerf watermark (Settings)

Voice edit

  • "Right Hand, label this framing day 3" → updates caption/phase tag · audit on save
  • "Flag this as damage" → adds red chip overlay · routes note to PM
  • No inline text field · voice-first edit per D-046

F-FD1 cross-ref

  • Single-capture detail (F-FD1) shows routing log + transcript · viewer stays lightweight
  • Drill to F-FD1 from viewer metadata link (deferred · "More detail" affordance)
What this surface enforces Business brain, not file cabinet. RH summary tells the operator what's on the job before they scroll 248 thumbs. Voice lands on the same surface with filter pre-set. Every capture ties back to a daily log — Media is the flat browse layer on top of F-E1's day-scoped canon.
What this surface refuses to be Not a folder tree. Not a List/Grid comparison toggle. Not an "extracted entities" panel. Not CompanyCam's project album with tags UI. F-ML1 is the project's visual memory — browse recent, voice-retrieve specific, drill to the daily log that owns the capture.
