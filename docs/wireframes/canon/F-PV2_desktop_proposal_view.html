4-question rule

  • Macro / comparison / detail? Detail · single artifact (this proposal · pre-send review)
  • List / grid / both? Neither · Grid Doctrine refused on detail/action surfaces (D-046)
  • Cognitive job? Operator reviews what the client will see · validates source chain · gates the send · keeps margin/provenance off the artifact
  • Hidden until drill? Source-chain artifacts (transcript · field notes · past projects) drill from rail · send confirmation modal surfaces on Send click

Canon distinction · why this surface exists separately from F-PS1

  • Proposal · unsigned · pre-contract · revisions modify it · client previews via F-PV1 (mobile) / F-PV2 (this, desktop)
  • Contract · signed proposal · post-signing · change orders modify it · status views via F-PS1
  • Change order · scope mod to executed contract · CO previews via F-B1c
  • F-PV2 has NO CO references because no contract exists yet — gate check #6 enforces this
  • Signature block at the bottom of the paper is the canonical pre-contract marker — "this proposal becomes a contract when signed"

Action bar · deviation from F-G1 trio (intentional)

  • F-G1 trio: Send back to draft · Save & continue editing · Approve & send (draft-review ergonomic · editing still in scope)
  • F-PV2 trio: Back to draft · Export PDF · Send for signature (pre-send preview ergonomic · editing already done at F-G1)
  • Why divergent: F-G1 is operator-internal scope/pricing assembly where save-and-continue matters. F-PV2 is the last-look preview before external send · editing here means going back to F-G1 (not staying on the preview). Export PDF is the new third action because operators often want to share the PDF with collaborators or print before triggering send.
  • Both surfaces gate-check before the destructive action · the trio shape matches the surface's cognitive job
