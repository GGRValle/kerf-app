F-TD1 · Mobile Global To-Do · cross-project task queue

Tier 2 cross-project queue. Reached from Home brain-question "What needs my attention?" or bottom-bar slot 4 when on this surface. Same lens-not-console discipline as F-PR1. F-TD1's primary cognitive job is "see every open task across jobs, sorted by urgency, and act or assign without opening each project." Not a project database. Not a kanban board. Right Hand summary leads, rows sort by due-date urgency (overdue → today → this week → later), entity chip shows where the task lives, voice add at top.

Frame 1 · F-TD1 · Global To-Do · List view · default · 18 items visible

Owner opens Global To-Do from Home or bottom bar. RH synthesis first — "18 to-dos total · 4 due today · 2 overdue · 6 projects with pending items." Filter chips default to All. Voice-add chip sits below filters. List sorts into five time sections with explicit counts: Overdue (2) · Today (4) · This week (6) · Later (4) · Done (2 shown + rollup). Each row carries task description, linked entity chip, due chip, assignee avatar, and state — red chip-tier reserved for overdue due chips only, never row borders. This is the GC archetype baseline (15-20 open tasks across 8 projects + standalone). At 50+ tasks the same compression rule applies: top items per section + "+ N more" rollup per section · Grid view is the per-project comparison override.

F-TD1 · Spec

Build position

Lane 5 · To-do system
✓F-PR1 · F-PR2 (Projects list + detail)
2F-TD1 · Global To-Do mobile (this)
3F-TD2 · Global To-Do desktop (next · designed as pair)

4-question rule

  • Macro / comparison / detail? Comparison · cross-project task queue (not per-job detail)
  • List / grid / both? Both · List default (time-sorted sections) · Grid alternate (per-project tiles · distinct chrome)
  • Cognitive job? See what needs doing across the whole shop · act by urgency · drill to the job that owns each task
  • Hidden until drill? Task detail sheet · F-PR2 project To-do sub-tab · filter-chip regroup modes (By project · By assignee)

Doctrine applied

  • d-1One primary cognitive job — see cross-project task queue, act by urgency
  • d-2Smart summary leads — RH synthesis line at top, not stat grid
  • d-3Sort by urgency — overdue → today → this week → later → done compressed
  • d-4Red chip-tier only — overdue flagged as due chip, not row border
  • d-5Voice add visible — quick-action chip + RH hint at top of list
  • d-6Filter chips for scan modes — All default · Today · Overdue · By project · By assignee · Standalone
  • d-7Grid = comparison surface — per-project tiles · distinct chrome from list rows

Row anatomy

  • Task description · 1 line · truncatable · primary text
  • Entity chip · project name · client name · vendor name · or "Standalone"
  • Due chip · Today · Tomorrow · Fri · 2d overdue · No due date · red tier for overdue only
  • Assignee avatar · initials · Owner (CA) · PM (MT) · Field Hand (JS) · unassigned (—)
  • State tag · Open · Done · Blocked
  • Arrow · drill-in to task detail or linked project

Section order (locked)

  • Overdue · 2 items · red due chip only
  • Today · 4 items · amber Today chip
  • This week · 6 items · day-name due chips
  • Later · 4 items · date or No due date
  • Done · 2 shown + + 2 more completed rollup · 4 total done

Filter chips (top bar)

  • All · active by default
  • Today · due today only
  • Overdue · past-due open items
  • By project · regroups list under project headers (deferred interaction)
  • By assignee · regroups by Owner / PM / Field Hand / unassigned
  • Standalone · items not linked to a project

Voice add

  • Quick-action chip: Add to-do with mic icon
  • Hint line: "Right Hand, add a to-do: …"
  • Top-right Add action duplicates typed entry
  • Completion via voice: "Right Hand, mark the Hernandez tile check done"

Scale lens · 15-20 baseline · 50+ stress test

  • GC archetype baseline: 18 items visible (16 open + 2 done + rollup) across 8 projects + 2 standalone + 1 vendor-linked
  • Projects represented: Wegrzyn · Hernandez · Patel · Goldberg · Ault · Foster · Nakamura · Cheng · O'Brien · Martinez · Dunne · Reyes (via standalone hardware)
  • Compression rule: when a section exceeds 6 items, show top 6 + "+ N more" rollup per section
  • 50+ stress test: same row anatomy · aggressive section rollups · Grid view shows all 8+ project tiles at once for comparison · filter chips become primary navigation

Bottom bar

  • repeat(5, 1fr) · 3+1+1 pattern
  • Slot 4 (Clients position) = To-do ACTIVE on this surface
  • 🎤 Speak FAB unchanged · routes to RH with to-do context

Routes

  • Back arrow → F-A1 Owner Home
  • Add / voice-add chip → create to-do sheet (voice or typed)
  • Row tap → task detail or linked F-PR2 project To-do sub-tab
  • Entity chip tap → F-PR2 project detail
  • Grid tile tap → F-PR2 project To-do sub-tab filtered to that project
  • Bottom bar To-do stays active · other tabs route to respective destinations

What this surface refuses to be

  • Not a kanban board with status columns
  • Not a Gantt or calendar replacement
  • Not a multi-field task database with 12 custom properties
  • Not Houzz Pro's checklist buried inside each project only
  • Not agent-named workflows · only Right Hand

Frame 2 · F-TD1 · Grid view · per-project to-do comparison · distinct chrome

Operator taps Grid in the view-mode toggle. Per-project tiles replace the time-sorted list — different chrome from List per comparison-surface doctrine. Each tile shows open count (large numeral) · project name · next due chip. Standalone and vendor-linked items get their own tiles. No time-section compression — Grid is the "show me workload by job" view. All 8 active projects + standalone represented. Drill into any tile for that project's To-do sub-tab in F-PR2. Pattern stress-tested at 50+: same tile shape · vertical scroll · 2-col mobile · filter chips narrow the tile set · overdue projects sort to top via red next-due chip inside tile.

F-TD1 Grid · Spec

Comparison surface canon

  • Distinct chrome from List · large count numerals · gradient tiles · grid-compare-bar header · no time sections
  • 2-column mobile · per-project + standalone + vendor tiles
  • Tile content: count · project name · client/sub · next due chip + task hint
  • Overdue projects carry red next-due chip inside tile · not row border
  • Drill → F-PR2 project To-do sub-tab filtered to that project
  • 50+ stress test: same tile · scroll · filter chips narrow set · sort overdue-first
What this surface enforces Cross-project task queue, not a checklist buried in each job. RH synthesis tells the operator what's due across the whole shop. Rows sort by urgency. Entity chips show where each task lives without opening the project. Voice add is one tap away. Grid view compares workload by project — different chrome from the time-sorted list.
What this surface refuses to be Not a kanban board. Not a calendar. Not a task database with custom fields. Not per-project-only to-dos with no global view. F-TD1 is the operator's unified action queue with drill affordances to each job. Red tier reserved for overdue chips only. Only Right Hand — no agent names.
