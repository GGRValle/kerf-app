Desktop density choices

  • 4-column row at desktop width · project name + phase | client + address | state | arrow. Mobile stacked all three into one block.
  • Right rail-led portfolio synthesis · mobile had RH summary above the list and portfolio footer below; desktop puts both in the rail so the list dominates the main column.
  • 2-week lookahead in the rail · 5 day-grouped rows. Same data F-A2 surfaces at Owner Home, scoped here to the active 4 projects.
  • PRJ ids visible · meta line carries the system id at desktop width without crowding (mobile dropped it).
  • Day count visible · "day 4 of 11" surfaces project rhythm where mobile collapsed to last-touch only.

Doctrine bound (same as F-PR1, scoped to desktop)

  • d-1One primary cognitive job — see portfolio state, drill where work is happening
  • d-2Smart summary leads — rail RH card opens the page
  • d-3Sort by attention — drift → pending → on-track → bid → closing
  • d-4Phase as text tag · status colors reserved for state (red drift · amber pending · cyan closing · green release)
  • d-5Portfolio summary in rail, not page header — reference content never leads
  • d-6No filter bar · default sort handles 80% · search via global chrome search
  • d-7Trust surfaces exception-tier — no transcript/audit nav on this page
  • canonRight Hand named once · no agent names elsewhere

Section order (locked · revised 2026-05-18 PM late+ per D-046 scale lens)

  • Active · projects in flight (sorted within: drift → pending → on-track) · ~6 jobs for GC archetype baseline
  • Closing · finishing this week or next · distinct from Closeout (delivery, not punch list)
  • Bid · in proposal/bid stage · feeds from Sales pipeline · shown here so Owner sees pipeline-to-handoff arc in one place (revised from prior "not shown for Owner" rule)
  • Closeout · punch list + retainer release · cyan accent
  • Warranty · post-handoff callbacks · low-frequency but kept in-view so they don't fall out of sight
  • Archive · long-press / Actions menu · not on default view

Scale lens · 15-20 baseline · 50+ stress test

  • GC archetype baseline: 20 active projects across the 5 sections (this surface's default sample)
  • Compression rule: when a section has >5 attention items, show top 5 + "+ N on-track" rollup (Active section here demos with 3 attention + 3 rolled up)
  • Service-trade stress test (50+ work items): compression rolls up aggressively (top 5 attention + "+ 42 on-track") · same row anatomy · Grid view is the "show me everything" override · section header click → expanded view
  • Desktop density advantage: 4-column row spreads name/phase + client/address + state + arrow across full width · mobile stacks into one block per row · same data, different density

Routes

  • New project (top-right) → F-S1 · Start sheet

What this surface refuses to be

  • Not a project database with 47 fields per row

  • Not a filter-and-sort dashboard with multi-select chips
  • Not a desktop "see more" expansion that buries the list under aggregate widgets

F-PR3 Grid · Spec

Big-picture canon applied

  • Comparison view · every active job visible at once
  • 4-column desktop grid · 12 tiles fit easily on one screen · no attention compression
  • Tile content disciplined: name + phase + client+address + state chip + rhythm line
  • Map view (V1.5+) shown as third toggle option (deferred)
  • Drill into any tile → F-PR4 Project Detail

What Grid view refuses to be

  • Not a kanban (no drag between status columns)
  • Not a dashboard with micro-metrics per tile
  • Not a Houzz Pro tile grid with 47 fields per card
  • Scan-first · disciplined per the big-picture canon
What this surface proves at desktop density The mobile list works at desktop without becoming a project database. Adding columns, not features. The third column (client + address) gets its own zone instead of cramming into the name line. The portfolio rail carries the RH summary and the lookahead — so the main column stays a list, not a dashboard. Wegrzyn drift surfaces twice (once as a row decoration, once as a rail callout) — that's intentional: it's the highest-attention item in the portfolio.
What this surface refuses to be Not a Gantt. Not a kanban. Not Buildertrend's project grid. Not a tile dashboard with custom-column filters. F-PR3 is F-PR1 with breathing room, plus the lookahead the rail can carry. A 1700px viewport spent on the same cognitive job — portfolio state at a glance — not on more widgets stitched together.
