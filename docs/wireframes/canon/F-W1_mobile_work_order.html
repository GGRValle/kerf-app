What's visible to Carlos (Field Hand permission scope)

  • Today's scope-anchored work (what to do)
  • Crew (himself + Tito)
  • Project name + address (where)
  • Expected vs logged hours on the scope line (how long)
  • What to capture (photos, voice notes)
  • What NOT to proceed on without checking (verify-before-release rule)
  • Clock state (clocked in since X, hours today)
  • Single quick-capture affordance → F-E1 Field Capture

What's HIDDEN from Carlos (permission lattice)

  • No pricing · scope-line dollar amount never appears · per F-31 canon and `project_kerf_blackboard.md` permission lattice
  • No margin · operator-private
  • No CO-002 pending state · that's owner/PM concern · Carlos sees scope as it currently is, not pending changes
  • No client communications · Carlos doesn't see Maria's tile-spec change message · that lives in PM/Owner Comms
  • No other crews' work · sees Tito (paired) but not Wegrzyn or Dunne deployments
  • No transcript review · F-F1 is exception-tier · Field Hand never lands there
  • No audit · same reason
  • No drift-management surface · that's PM's lane

The "don't proceed without checking" doctrine

  • LiDAR use-label rule visible: shelving wall length is estimate-safe (room scan), NOT fab-safe (tape required)
  • Per kerf_lidar_precision_limits_and_apple_wrap.md · cabinetry / stone / glass / millwork blocks release until tape-verify logged
  • Surface tells Carlos: "Marcus will assign a verify task once framing is closed" — the workflow continuation is named, the crew doesn't have to figure it out
  • This is the field-hand-side instantiation of trust-first-precision-later (`feedback_trust_first_precision_later.md`)
