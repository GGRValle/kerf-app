F-S1 · Intent-first canon · what changed and why

Prior version (superseded): 5 capture-verb cards — Add client/lead · LiDAR scan · Voice/video walk · Photo/video · Plan/document upload. Media-led. Asked the user to pick which tool / file type they wanted to produce.

This version: Two sections of intent-led starts. "Start work" = 5 business actions (field walk · change order · estimate/proposal · client intake · takeoff). "Start a sweep" = 3 agent workflows (clarify · lookahead · drift check). User picks the action; Kerf chooses the medium on the next step.

Why: Per kerf_start_action_shortcut_canon.md. Media-led Start was one tap from drifting into a Houzz Pro launcher grid. Intent-led keeps the surface on doctrine — capture reality via what the user is starting, not via the file type they're producing.

  • Start field walk → opens voice capture with photo/video add-on; routes capture into the active project's Daily Log
  • Start change order → voice-first interview pre-bound to the active project; photos and measurements optional supporting evidence
  • Start estimate / proposal → voice intake; plan/document upload; LiDAR scan if room measurements needed
  • Start client intake → form (default) · voice (verbal walkthrough) · email-import (paste a client inquiry); creates a Sales-stage record or a Clients record depending on phase
  • Start takeoff → opens LiDAR scan flow directly (the natural medium for takeoff); pre-bound to current project context if applicable
  • Start clarify → picks an open ambiguity from the Holding rail (scope item with confidence flag, transcript edit pending review) and surfaces a resolution interface
  • Start lookahead → generates a 2-week schedule projection across active jobs; flags conflicts before they happen
  • Start drift check → sweeps active jobs for schedule + scope conflicts on demand (vs. the background drift detection that runs continuously)

Verb-not-agent enforcement: Every label is plain-English work. "Start clarify" not "Start clarification agent." Sub-descriptions name the medium or the work, never the agent. Per feedback_agent_names_not_in_operator_copy.md.

Visual conventions: Start-work items use amber accent icons (matches the Owner accent + signals "actions you take"). Start-a-sweep items use violet accent icons (matches existing canon-violet usage for informational/computational content and signals "the system does something for you"). Both stay within the 3-color status discipline since these are role/category accents, not status colors.

What this surface refuses to be: Not a launcher grid. Not a "pick a media type" prompt. Not an agent picker. Not a Houzz Pro create-new-something hub. Start is the verb the contractor speaks; Kerf carries the rest.
