F-US1 · User settings — your login, not the company's

The settings any logged-in user reaches by pressing their name/avatar (upper-right on desktop, header on mobile). It's just them — profile, login & security, personal preferences. It is deliberately separate from Admin settings (team · permissions · billing · integrations · audit), which is a higher-permission surface. A field hand has F-US1; only an owner/admin also has Admin. Supersedes the old F-SP1a account editor (and adds the theme toggle it was missing).

This is just you. Company-wide settings — team, permissions, billing, integrations, audit — live in Admin settings → (owner/admin only).

Personal · F-US1 (this surface)

  • Every logged-in user has it — owner, PM, admin, field hand, sub.
  • Scope: their own profile · login email · password · 2FA · sessions · language · theme · notifications.
  • No permission needed — it's yours. No tenant/company data, no other users.
  • Reached by the avatar/name menu → "Your account." Same content mobile + desktop.

Admin · F-SP1 (separate, higher-permission)

  • Only owner/admin roles. Hidden from the menu for everyone else.
  • Scope: team & users · permissions/roles · integrations · billing · Plays · navigation · audit — tenant-wide consequence.
  • Changing your password = personal. Changing someone else's access = admin. That's the line.
  • Tenant/company config (brand, defaults, F-DL3 guidance level) sits inside Admin (owner-level subset).

Notes

  • Supersedes F-SP1a (old account editor) — and adds the theme toggle it was missing.
  • Right Hand gold = personal/identity accent.
  • Security: password/2FA/email changes are real auth actions — the surface shows the control; the actual change runs through the auth flow with re-verification, never stored inline. (Wireframe note for the build lane.)
  • Title/role is read-only here (set by admin) — a user can't promote themselves.
