Frame 2 · Scale stress test

Compression rule

  • Recently run: max 8 visible rows · then dashed + N more rollup
  • Same rule as F-PR1 Active section · business brain not file cabinet
  • RH summary carries aggregate count · operator never counts rows manually
  • Category grid density unchanged at 50+ · drill for full inventory

What holds at scale

  • 6 category cards · fixed navigation anchors
  • 3 scheduled rows · rarely exceeds 5 in practice
  • Per-row Run · Print · Export actions · same touch targets
  • No List/Grid toggle introduced at scale · would violate grid doctrine
What this surface enforces Run the report, don't manage a report library. RH summary tells the operator what's available and what's stale. Category cards route to report families without exposing 42 individual definitions on one screen. Recently run + scheduled carry the actions operators repeat daily. Every export writes an audit trail entry — data egress is never silent.
What this surface refuses to be Not a BI dashboard. Not a query builder. Not Houzz Pro's export buried three menus deep. Not a Grid comparison of 50 report tiles. F-RP1 is the printable cross-cutting reporting home under Team & Ops — category navigation, recent history, scheduled jobs, and explicit Run · Print · Export on every row.
