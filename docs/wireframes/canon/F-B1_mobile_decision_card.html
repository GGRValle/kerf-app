Layered disclosure

  • Layer 1 Glance — what shows above: amount + 4 plain-English source items + trust passed
  • Layer 2 Tap — tap trust banner ➜ expand inline with 4 specific checks (source refs / pricing / role visibility / approval) in plain language
  • Layer 3 Full detail — tap "View audit detail" ➜ F-H1 Audit Detail (settings-located substrate)

Chrome contract

  • Top: back nav + centered title + space (sub-surface; not a primary nav destination)
  • Bar at bottom: Home active (came from One Thing tap)
  • If entered via Projects > Scope > CO, Projects would be active instead
  • Voice FAB still available · long-hold opens RH conversation about this decision

Action hierarchy

  • Preview client view · neutral · full-width above primary · opens F-B1c (PDF-style change order with itemization control)
  • Primary: Approve → · full-width · amber dominant
  • Secondary: Edit ↩ · Reject · Ask more · 3-up row, smaller, neutral
  • Reject styled with red text accent · still neutral background (don't make rejection feel destructive)
  • Approve commits scope/pricing/terms · routes to client-projection queue · does NOT auto-send (V18 gate next)

Preview affordance (carries over from F-B1b)

The owner can see the client view before editing — useful when the operator wants to sanity-check whether the as-built draft already reads well to a client. Preview from F-B1 = same destination as Preview from F-B1b (F-B1c). Itemization level (Detailed / Summary / Lump sum) and Show-margin toggle live on the preview screen.

Routes

  • Back arrow → F-A1 Home (or wherever drilled from)
  • Trust banner tap → expand inline (Layer 2)
  • Source banner tap → F-35 Draft Review (the underlying draft)

  • Edit → return to F-35 Draft Review in edit mode
  • Reject → write reject decision, return to Home
