F-PR2 · Mobile Project Detail · one job, nine lenses

Per-project lens. Reached by tapping a project from F-PR1 or via Client Detail's project card. Four guardrails applied

  • G1Lead with project state synthesis. Header + RH summary + 5 explicit state rows render BEFORE the tab strip. Operator sees Phase · Moving · Blocked · Decision · Changed at a glance, in 5 seconds. The tabs render below that.
  • G29 tabs feel like lenses into ONE job. The RH summary at top names what each tab will carry: schematic (Scope) · framing day 2 (Schedule + Field) · CO-002 awaiting approval (Scope + Budget) · field walk captured (Field + Comms) · media library (Media) · project to-dos (To-do). Each tab is a different angle on the same project. The strip is visually secondary.
  • G3Field / Schedule / Budget continuity visible. The CO-002 card on the Scope tab carries an explicit "Built from" callout: Field walk → transcript → priced against your past pantry jobs. The chain reads on the surface — operator sees field reality flowing into scope into money without leaving the page.
  • G4Audit available, not loud. Audit is the 9th tab. It renders at equal visual weight to the others — but with no count chip (other tabs have counts: Schedule 3 · Field 2 · Media 327 · Comms 5 · To-do 10 · Files 4 · Scope CO flag). The absence is the quiet. Operator only clicks Audit when something needs verification — which matches the exception-tier reclassification.

5 state signal rows (the canonical questions)

  • Phase · what stage the project is in
  • Moving · what's happening today (with on-track/drift state)
  • Blocked · what's stopped (or "nothing right now" — italicized muted text)
  • Decision · what needs operator approval/action (amber when pending)
  • Changed · most recent material event

Each row is plain English + state. Color-coded only when action is implied. Not a stat grid.

Tab strip · 7 lenses

  • Scope (default) · proposal · COs · exclusions · allowances
  • Schedule · tasks · milestones · crew assignments · lookahead
  • Budget · estimate vs actual · CO impact · allowances spent · margin (Owner-only)
  • Field · daily logs · captures · transcripts (read-only via deep-link)
  • Media · all project photos/videos/daily logs · flat browse · drills to F-ML1 (Lane 6)
  • Comms · client emails · SMS · portal threads · designer messages
  • To-do · project-scoped tasks · assignee · due · stage · voice add · converts to punch at closeout (Lane 5 · see Frame 2)
  • Files · drawings · permits · signed agreements · photos-as-deliverables
  • Audit · per-project decision log · filtered view of Settings → Audit

Tab count semantics

  • Scope shows "CO" flag (amber) when a CO is in flight · otherwise no chip
  • Schedule shows count of milestones in next 2 weeks
  • Budget · no chip (single state, no count)
  • Field shows count of recent captures (last 7 days)
  • Media shows total media count (photos + videos + daily log entries with media)
  • Comms shows count of recent threads (last 14 days)
  • To-do shows count of open project tasks
  • Files shows total file count
  • Audit · no chip · the absence is the visual quiet (guardrail #4)

Continuity callout (guardrail #3)

The CO-002 card carries an explicit "Built from" line that names the chain in plain English:

  • Field walk (Field domain) →
  • Transcript (Field domain, exception-tier access) →
  • Priced against your past pantry jobs (Sales/Budget domain via comparables)

"View chain →" link expands the continuity into a stepped diagram (out of scope for this build · sketches in spec). The point: operator sees Field reality flowing into Scope decision into Budget impact without leaving Project Detail.

Scope tab content order

  • Proposal (compact card) · single line summary of the base scope
  • Change orders (most prominent · CO-002 is the action) · in-flight CO card with continuity callout
  • Exclusions (compact list · 3 items)
  • Allowances (compact list with $ amount + $ spent)

Each section is glanceable. Scope tab is dense but appropriately so — this is the consequence center.

What this surface refuses to be

  • Not 9 mini-dashboards stitched together
  • Not a Gantt chart
  • Not a project management workstation with toolbars and bulk actions
  • Not Buildertrend's tab-of-everything-per-project
  • Not a contact-style profile page
  • Not a "click + and fill out 47 fields" project creator
  • Not an audit forensics console (Audit tab is quiet · forensics live at F-H1)

  • canonTrust surfaces stay exception-tier · Audit is the 9th tab with no count, operator clicks only when needed · CO-002 trust check passed framed in plain English
