F-LIB1 · Libraries + Selections — the catalog, and the job-specific instances

The catalog/instance split (D-060 §3). Frame 1: the Libraries area — Cost · Selections · Vendors — with the Selections Library open: tenant-level reusable reference data. Library is the menu; Project Selections are the order.

Frame 1 · Selections Library

What it is (D-060 §3)

  • Tenant-level reusable catalog — products, finishes, fixtures, materials.
  • Each item carries spec, image, typical price, vendor, lead time, "used in N jobs."
  • Tenant-private — your shop's catalog, grows over time. Sibling object to the Cost Library.

Libraries area

  • Groups reusable reference data: Cost · Selections · Vendors.
  • Accessed from Design + the estimate builder; managed here as libraries (one place, like the Connections registry).

It is not a domain

  • Reference layer. The job-specific record is the Project Selection (Frame 2).

Proposed

Instances, not catalog: each row lives on Moore and points back at a Selections Library item ("from Library"), but carries its own lifecycle — proposed → approved → ordered → installed — plus its allowance, its PO, and (once installed) its actual. Approving one writes a PO to the vendor, reconciles the allowance in Money, and drops an install line on the Schedule. "Add from Library" pulls the catalog; anything spec'd here that's new can be saved back to the Library.

Frame 2 · Project Selections

Instance record (D-060 §3)

  • Lives on the Project; references a Library item (library_ref).
  • Lifecycle: proposed → approved → ordered → installed.
  • Carries its own allowance · PO · schedule tie · actual.

Propagation

  • Approve → PO to vendor · allowance reconciliation in Money · install line on Schedule.
  • Estimate line types (material/product/equipment/subcontract) promote to a Project Selection.
  • New item spec'd here can be saved back to the Library.

Rules carried

  • Back control (non-home, D-060 §1). Money writes stay confirm-gated.
