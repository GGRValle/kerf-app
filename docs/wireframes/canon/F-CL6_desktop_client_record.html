Why "canonical record"

F-CL5/CL6 are the authoritative client entity surface. A lawyer reviewing a dispute, a bookkeeper reconciling AR, a future system migration extracting client state — all should consider this view source-of-truth. F-CL4 was experience-led; canonical record demands completeness without becoming console-y. Business brain leads, file cabinet sits underneath.

Four passive baseline sections (lightened Overview)

  • Projects · short list of jobs (active project named with risk chip if any · closed rolled up · drill to F-PR4 for active work)
  • Open balance · single AR row with aging chip · drill to Money tab for invoice ledger · payment history · margin posture
  • Last touch · single row · operator-readable timing of last interaction
  • Source · single row · referral / inquiry origin · LTV multiplier if known

Above the four sections: Header (name · since · LTV · address · CLI-006 · primary contact) + RH synthesis sentence. Right rail keeps Contact + Money snapshot (2 rows: LTV + AR open) + "What I'm watching for you" — upcoming touches + anniversaries only · not active monitoring (that lives at Project detail). Per D-046 §C2 lighten directive — supersedes the prior 5-question "developed document" treatment.

Six sub-tabs · drill-down lenses

  • Overview (default · this render) · synthesis page
  • Projects (2) · 1 active · 1 closed-won · routes to F-PR4
  • Comms (14) · text · email · portal threads · last 14 days
  • Notes (7) · operator notes · voice transcripts · RH summaries
  • Files (22) · proposal · contract · LiDAR scans · permit · photos · invoices
  • Money · AR · payment history · margin per job · LTV breakdown · no count chip

Layout · main + rail · desktop projection of the lightened mobile sections

  • Main column · the four passive sections vertically (Projects → Open balance → Last touch → Source) · each compact, single-row or short-list shape
  • Right rail · Contact card → Money snapshot (2 rows: LTV + AR open · drill to Money for breakdown) → "What I'm watching for you" (upcoming touches + anniversaries only · RH-toned)
  • Mobile vertical stack becomes desktop main+rail · same content, denser arrangement · same passive-baseline hierarchy
  • Watching card promoted to rail at desktop (was Overview-bottom on mobile) — visible without scroll

Grid doctrine applied

  • DETAILThis is a detail surface — single client depth. No grid view. No view-mode toggle.
  • Single-frame wireframe per the 4-question dispatch
  • Sub-tabs are lenses into one client, not parallel views of the same surface

Disciplines applied

  • D1Business brain, not file cabinet — synthesis leads · Files tab one click deeper
  • D2Red is chip-tier only — "At risk" chip in active project card head; "11 days" inline chip on AR row. No full-row red borders.
  • D3No agent names in operator copy — "Tile sub conflict flagged" / "AR aging 11 days" — never "Drift Agent flagged" / "Money Agent alerted." Only Right Hand named (RH eyebrow on synthesis + Watching card).
  • D4Translate canon vocab — "your past jobs" · "what I'm watching for you" · "history snapshot." No cohort/altitude/P25–P75/AltitudePacket language at the operator surface.
  • D5Clients live under Team & Ops — sidebar shows Team & Ops parent active with Clients sub-item active, per kerf_client_section_canonical.md demotion canon.

What this surface refuses to be

  • Not a Salesforce contact workstation with 40 editable fields
  • Not a Houzz Pro "client folder" experience (folders sit underneath, brain leads)
  • Not a HubSpot-style activity feed dominated by automated touches
  • Not a comparison grid of clients (that's F-CL3)
  • Not a marketing engagement-scoring dashboard
  • Not an audit forensics console (Audit lives at Settings → Audit)
  • Not a "client = 1 contact" CRM model — handles primary + spouse + future multi-contact
