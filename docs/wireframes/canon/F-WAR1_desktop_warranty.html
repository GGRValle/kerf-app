Frame 1 · Spec

Client Success entity (D-060 §10)

  • Warranty is a post-handoff obligation — it lives in Client Success, tied to the closed-out project.
  • Holds terms, covered components (workmanship + mfr/fabricator pass-through), dates, and the claims log.
  • "File a claim" → a tracked claim; Right Hand can open one from a client message or call.

Ties

  • Covered components reference the Project Selections / subs that did the work.
  • A claim can spawn a work order to the responsible sub (two-sided, F-SUBM1).
  • Sidebar reflects drill-down: Client Success → Warranty (§13).

Same entity, two views. The warranty record lives in Client Success (Frame 1) where it belongs — it's about the relationship after handoff. Admin sees the tracking projection (Frame 2): what's active, what's expiring (a Client Success outreach trigger — "your warranty's ending, here's a tune-up"), and where claims are open. An expiring warranty is an Attention Artifact on both the Admin and Client Success homes.

Frame 2 · Spec

Admin tracking projection

  • Read/track view over the Client Success warranty entity — not a second copy.
  • Filters: All · Active · Expiring 90d · Open claims.
  • Expiring → Attention Artifact + a Client Success outreach moment (renewal / re-engagement).

Placement

  • Reached from Admin/Ops (Money). Entity reached from Client Success. Back control on both (§1).
  • Consistent with Subs: one entity, role-specific projections.
