Desktop logic preserved

  • Same mental model as F-A1 mobile · One Thing / Business state / On Deck / Pulse
  • Same bar contract translated to sidebar — Home (mandatory) · Projects (swap) · Speak (mandatory, FAB) · Clients (swap) · Settings/Audit (under More function, here exposed directly)
  • Denser presentation — all 9 domains always-visible in sidebar because density allows. Mobile compresses 4 of them under More; desktop doesn't need to.
  • "More" function becomes the bottom group (Settings · Audit · Tier C hook) rather than a sidebar opener

Sidebar markers (legend)

  • MMandatory anchor (Home · Speak)
  • SUser-defined / swappable (Projects · Clients · default)
  • VAlways-visible secondary (Sales · Schedule · Field · Money · People · Client Success)

Right Hand summary line

  • New element on desktop (mobile lacks vertical room for it)
  • 1-line synthesis: active jobs · pending decisions · drift · pipeline · AR state
  • Speaks through Right Hand: "one decision needs you", "one drift to handle"
  • Sits under greeting · above One Thing card · sets context

5-cell business state

  • Mobile shows 4 brain-question cells; desktop room enables the 5th (Pipeline)
  • Each cell has sub-line of detail (mobile hides this; desktop shows it)
  • Cells: Where are my guys / Where is my money / What's closing or drifting / Client health / Pipeline opportunities

2-column grid

  • On Deck (left) — 5 items vs mobile's 3
  • Pulse (right) — 8 items vs mobile's 4, spans yesterday + today
  • Right rail is wider (1.2:1) to give Pulse more breathing room

Pulse grammar

  • Same as F-A1: event · object · consequence
  • Color-coded consequences: green = forward · amber = needs attention · violet = informational · cyan = revenue/growth signals (new on desktop given Pulse depth)
  • Yesterday's items shown with "Thu" prefix on timestamp
