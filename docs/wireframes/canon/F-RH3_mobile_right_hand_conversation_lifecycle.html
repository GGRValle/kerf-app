F-RH3 · Right Hand conversation · how it starts, how it ends

The explicit start-to-end of one Right Hand turn — built to answer Codex's fair question ("when the conversation stops, what do I do with the info — where does it go?") without leaving the surface. The whole turn happens on one persistent, blurred surface. The mic never goes away. Stop never swaps to a new card. The "where does this go?" confirmation appears as a bubble inside the conversation — and only at stage 4, because that's the moment a durable write is about to happen.

The contract (read this first, Codex):  ① background blurs, conversation lives in one growing surface · ② the same mic stays available the entire time · ③ Stop appends Right Hand's reply in place — it never mounts a new screen or a 3-button card dialog · ④ a confirm affordance appears only when filing / sending / money / a durable write is one tap away, and it's phrased as an answer to the real question (Save to Wegrzyn? · Change · Keep talking) — not a generic Save/Don't-save/Keep-talking box · ⑤ "Saved" only renders after the write returns · ⑥ the composer grows with what you type/dictate (caps at a few lines, then scrolls), pushing the conversation up so you can read it form · ⑦ paste & attach are first-class — drop text, a photo, a plan/spec/PDF, or an existing estimate in and Right Hand reviews it (reversible, no confirm); turning it into a draft/file/send is the consequence point. Applies to mobile AND desktop.

How it starts (1–2)

  • Tap the mic anywhere → background blurs, the conversation surface rises over it. The screen you were on stays visible behind, blurred — you never "left."
  • Live caption streams your words as you speak (realtime transcribe). Mic is hot, waveform moving.
  • Honesty up front: "I'll ask before I file anything."

How it ends (3–5)

  • Stop = reply in place. Your words settle, Right Hand answers in prose in the same surface. No new screen. No 3-button card. Mic stays — you can keep talking.
  • Right Hand resolves the turn (intent + where it belongs) and says it back plainly.
  • The conversation panel just grows taller as it fills.

The consequence bubble (4) — Codex's answer

  • The "what next / where does this go?" confirm is a bubble in the conversation, appearing only at stage 4 — because a durable write is about to happen (file / send / money).
  • It's phrased as the real question: "Save to Wegrzyn → Daily Log? Save · Change job · Keep talking." The affordances answer that question — not a generic Save / Don't-save / Keep-talking dialog.
  • Change re-routes (different job) or edits inline; Keep talking stays in the turn; the mic is still there.
  • Stage 5: Filed · via voice renders only after the write returns (no false "Saved"), and the surface stays open for the next move.

The composer — grows + paste/attach for review (mobile & desktop)

  • Auto-grows with typed/dictated input (caps ~6 lines, then scrolls internally); the conversation slides up so you read it form. Voice + type coexist in the same composer.
  • Paste / attach is first-class — text, a photo, a plan/spec/PDF, an existing estimate. It's the warm-input path complementing cold capture (voice/camera) — the heart of capture-to-quote ("here's the plan, build me an estimate").
  • Review is reversible (no confirm): "I read the plan — here's the scope, 3 gaps." Drafting/filing/sending from it is the consequence point (the green bubble).
  • Evidence, not truth (like crew SMS, D-052): Right Hand reviews & proposes, never auto-acts. Tenant-scoped; validator-walled (no model-written money; every extracted claim carries a source ref). An attachment lands as a job document (like a Scan) with an honest via paste / via upload chip.
  • Same on desktop (the right-docked panel composer) and mobile (the overlay composer).
