F-LND1 · Spec

What the login does

  • Email + password or SSO (Google / Microsoft 365) — real auth, not mocked in this wireframe.
  • Sign-in is always to a workspace (the tenant). Switching workspace is explicit — no cross-tenant blur.
  • After auth, Right Hand routes you to your role home (Frame 2).

Safety (locked rules)

  • Right Hand never enters or stores your password — credential handling is the auth layer's job, not the agent's.
  • SSO consent is a real OAuth grant the user approves; the agent doesn't auto-accept it.
  • Workspace = tenant boundary. A login resolves to exactly one tenant; isolation starts here.

Personal vs. admin

  • Personal settings (your email, password, language, theme) → F-US1, reached from your name top-right.
  • Tenant / admin settings (users, permissions, connections) → Settings in the sidebar, higher permission.

Routing · Spec

Role → home

  • Role lives on the user account (set by the workspace admin), not chosen at login.
  • One person can hold more than one seat; Right Hand lands them on their primary and lets them switch.
  • All homes resolve to the same path / — the projection is server-side by role, not a different URL.

Platform

  • Desktop homes: F-A2 · F-P2 · F-AO2 · F-TO2 · F-SU2 · F-ES2.
  • Field Hand is phone-first (F-FH1); Sub (F-SH2) is the same tiny projection on either device.
  • Mobile versions of the office homes are the next mapping pass.

Open

  • Role consolidation (PM/Super/Estimator/Admin-Ops own home vs. fold) feeds straight into this table.
