F-CL1 · Mobile Clients list · Tier 2 proving ground

First Tier 2 surface. Reached by tapping Clients in the Owner default bar. Tests whether the doctrine locked in the visual review + reclassification holds when the natural temptation is to build a CRM forensics console. Per feedback_business_brain_not_file_cabinet.md: operator surfaces lead with smart summaries; folders sit underneath. Per the visual-review preface: "A surface should do one primary cognitive job." F-CL1's primary cognitive job is "see the state of my client portfolio at a glance and drill where attention is needed." Not a contact-management database. Not a filter-and-sort console. Not a CRM workstation.

Doctrine applied: Right Hand summary leads (synthesis line, not stat grid). Client rows sorted by attention-needed first (risk · pending · active · leads · repeat) — not alphabetical. Each row carries a state one-liner that tells the operator what's happening with this client right now, color-coded only when action is implied (amber pending · red drift · green happy). Phase indicator as plain text label, not a colored pill (status colors stay reserved for state). Pipeline summary at the bottom (funnel counts) — not a top-of-page header. No filter bar, no sort controls, no transcript or audit tabs. The surface refuses to be a forensics console.

F-CL1 · Spec

Build position

Tier 2 build · entry point
✓Tier 1 · 13 surfaces complete

The doctrine in action

  • d-1One primary cognitive job — see portfolio state, drill where needed. NOT a database browser.
  • d-2Smart summary leads — RH synthesis line at top, not stat grid, not filter bar
  • d-3Inline state, not side panel — each client row carries its own state one-liner; no side rail with extracted detail
  • d-43-color status discipline preserved — amber pending · red drift · green happy/referral · neutral for "on track / no action needed"
  • d-5Phase as text, not status color — Active / Lead / Qualified / Repeat shown as plain text label (don't conflate phase with state-needs-action)
  • d-6Pipeline at bottom — funnel summary is reference content; doesn't lead the page
  • d-7No transcript or audit nav — trust surfaces are exception-tier, reached on demand from artifacts

Row anatomy

  • Client name · primary text · scannable
  • Phase + project context · sub-line, plain English ("Active · pantry conversion" or "Lead · kitchen remodel inquiry")
  • State one-liner · what's happening with this client right now · color-coded only when action is implied
  • Left-border accent · amber for pending · red for risk · green for happy / referral signal · neutral border for "on track"
  • Arrow · standard drill-in affordance to F-CL2 Client Detail

Sort order (locked)

Rows sort by attention-needed first, not alphabetical:

  • Within Active: risk (red) → pending (amber) → on-track (neutral) → closing (cyan)
  • Within Leads: stuck/quiet (amber) → recently-active (neutral)
  • Within Repeat: referral-path (green) → no-active-touch (neutral)

Operator can override via long-press header → sort picker (alphabetical / project-count / last-touched). Default is attention-needed.

What this surface refuses to be

  • Not a contact-management database
  • Not a CRM-style multi-tab workstation
  • Not a filter-and-sort forensics console (default sort is attention-needed; explicit sort picker for everything else)
  • Not a "click + to add 47 fields" intake form (Add routes to F-S1 Start sheet → Start client intake; lean intake flow)
  • Not Houzz Pro's grid of contact cards with field-by-field detail

Routes

  • Back arrow → F-A1 Owner Home (or whatever destination drilled from)
  • Add → F-S1 Start sheet → Start client intake (intent-first; medium chosen on next step)
  • Client row tap → F-CL2 Client Detail (5 sub-tabs: Projects · Comms · Notes · Files · Relationships; Satisfaction appears when CS agent has signals)
  • Pipeline card tap → F-Sales pipeline funnel (Tier 3 surface — defer until built)
  • Bottom bar Home / Projects / Speak / More → respective destinations
  • 🎤 Speak (long-hold) → RH conversation pre-bound to Clients context · "tell me about my clients" / "what's at risk"

What's not on this surface (deliberately)

  • No filter bar (sort defaults handle the 80% case; long-press for the 20%)
  • No search bar at top (sub-surfaces don't need their own search — global search lives at app level)
  • No tabs (no Active/Leads/Repeat tabs — sections in one scrollable view instead)
  • No "View Pipeline" CTA up-top (pipeline IS at the bottom)
  • No "Inactive" or "Archive" section (those are filter views via long-press)
  • No designer/architect/referral-source filter (those live on Client Detail Relationships sub-tab)
  • No engagement metrics dashboard (that's Client Success domain, surfaces at 10+ tier)

Doctrine bound

  • canonPlain English · Right Hand framing once at top · no agent names elsewhere
  • canon3-color discipline (amber / green / red) reserved for state · phase is plain text
  • canonPermission lattice · Owner sees full; PM scope will hide margin-tied state; Field/Sub scope will not surface Clients destination at all
  • canonOne primary cognitive job · see portfolio state · drill where attention is needed
  • canonTrust surfaces stay exception-tier · no transcript/audit nav here
  • canonSmart summary leads · folders sit underneath · per feedback_business_brain_not_file_cabinet.md

  • Tile content disciplined: name + category + state chip + rhythm line · scan not detail
  • Map view (V1.5+) would be third toggle option · deferred
What this surface proves The doctrine holds at Tier 2. Smart summary leads · folders underneath works exactly the same at the Clients destination as it does on Home. The temptation to build a CRM forensics console was real (Houzz Pro and every CRM does it); the discipline refuses. Wegrzyn-at-risk reads in 5 seconds; Hernandez CO-pending reads in 5 seconds; the operator drills where attention is needed. That's the job.
What this surface refuses to be Not a contact database. Not a CRM workstation. Not a "click + and fill 47 fields" intake form. Not a filter-and-sort dashboard. F-CL1 is the portfolio-state synthesis with drill affordances underneath. The 80% case (see state, drill) handled by default; the 20% case (find a specific client by name) handled by global search at app level.
