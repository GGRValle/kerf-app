Why "canonical record"

F-CL5/CL6 are the source-of-truth surface for the client entity — what a lawyer, bookkeeper, or future system migration would treat as authoritative. F-CL2/CL4 (the earlier Client Detail) were experience-led variants; the canonical record framing demands completeness without becoming console-y. Business brain, not file cabinet.

Four passive baseline sections (lightened Overview)

  • Projects · short list of jobs (active project named with risk chip if any · closed rolled up · drill to F-PR2 for active work)
  • Open balance · single AR row with aging chip · drill to Money tab for invoice ledger · payment history · margin posture
  • Last touch · single row · operator-readable timing of last interaction
  • Source · single row · referral / inquiry origin · LTV multiplier if known

Above the four sections: Header (name · since · LTV · NPS · referral history) + RH synthesis sentence. Below: Right Hand "What I'm watching for you" — upcoming touches + anniversaries only · not active monitoring (that lives at Project detail). Per D-046 §C2 lighten directive — supersedes the prior 5-question "developed document" treatment.

Six sub-tabs · drill-down lenses

  • Overview (default · this render) · synthesis page
  • Projects · 2 jobs · 1 active · 1 closed-won · drill into per-project F-PR2/PR4
  • Comms · text · email · portal threads · last 14 surfaced
  • Notes · operator-written notes · voice transcripts · RH summaries (7 notes)
  • Files · proposal · contract · LiDAR scans · permit · photos · invoices (22 files)
  • Money · AR · payment history · margin per job · LTV breakdown

Grid doctrine applied

  • DETAILThis is a detail surface — single client depth. No grid view. No view-mode toggle.
  • Single-frame wireframe per the 4-question dispatch · Home = glimpse · Domain list = comparison · Detail = depth · Grid = macro scan · List = action-ordered scan
  • The sub-tabs are lenses into one client, not parallel views of the same surface

Disciplines applied

  • D1Business brain, not file cabinet — Overview synthesizes; Files tab is one click deeper. The four passive sections (Projects · Open balance · Last touch · Source) + header + RH synthesis + Watching card render before tabs become reachable.
  • D2Red is chip-tier only — "At risk" inside the active project card is a small chip. "11 days" on the AR row is an inline chip. No full-row red borders.
  • D3No agent names in operator copy — "Tile sub conflict flagged" not "Drift Agent flagged"; only "Right Hand" appears (on the RH summary + Watching card).
  • D4Translate canon vocab — "your past jobs" · "what I'm watching for you" · "history snapshot." No cohort/altitude/policy language at the operator surface.
