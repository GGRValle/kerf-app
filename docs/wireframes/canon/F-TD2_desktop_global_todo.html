4-question rule

  • Macro / comparison / detail? Comparison · cross-project operator queue at desktop density
  • List / grid / both? Both · 6-column table (List) · 4-column per-project tiles (Grid · distinct chrome)
  • Cognitive job? Scan what needs doing across all projects · patterns + per-day rollup on rail · drill to project context
  • Hidden until drill? Per-project filtered list · F-PR4 To-do sub-tab · voice capture sheet · Done-section rollup

IA decision (locked)

  • To-do is top-level under All — same tier as Projects, Schedule, Money. Not a sub-item under Projects.
  • Mobile bottom bar slot 4 = To-do when on this surface (F-TD1).

Desktop density choices

  • 6-column table · Task · Project · Due · Assignee · State · Action. Mobile stacked into card rows.
  • Rail-led synthesis · RH summary + patterns flagged + per-day rollup. List dominates main column.
  • Filter chips · All / Today / Overdue / By project / By assignee / Standalone — same set as F-TD1.
  • Voice add · breadcrumb bar chip with mic + hint text. Sidebar SPEAK FAB also visible.

Doctrine bound

  • d-1One primary cognitive job — see what needs doing across the business, drill to project context
  • d-2Smart summary leads — rail RH card opens the page
  • d-3Sort by attention — overdue → today → this week → later → standalone → done (compressed)
  • d-4Red chip-tier only — overdue as chip/text, not row border
  • canonRight Hand named once · no agent names elsewhere
  • canonList + Grid toggle on comparison surface only

F-TD2 Grid · Spec

Comparison view canon

  • Per-project tiles · open count + next due + client line
  • 4-column desktop grid · scan by project, not by individual task
  • Red/amber chips carry attention flags — no row border treatment
  • Drill into tile → filtered list for that project
  • Standalone items get their own section (not forced into project grid)

What Grid refuses to be

  • Not a kanban board with drag-between-columns
  • Not a Houzz Pro task grid with 47 fields per tile
  • Not a dashboard with micro-metrics per project
What this surface proves at desktop density F-TD1 works at desktop without becoming a task manager. Adding columns, not features. The 6-column table spreads task context across full width. The rail carries RH synthesis, patterns, and per-day rollup — so the main column stays a queue, not a dashboard. Hernandez surfaces twice (pattern card + 4-open grid tile) — intentional highest-attention signal.
What this surface refuses to be Not Asana. Not a Gantt-linked task board. Not Buildertrend's to-do module. F-TD2 is F-TD1 with breathing room, plus the patterns and rollup the rail can carry. A 1700px viewport spent on the same cognitive job — what needs doing across the business — not on more widgets stitched together.
