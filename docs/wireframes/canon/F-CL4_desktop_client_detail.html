Three guardrails preserved

  • G1Lead with client state, not tab strip. Client header + synthesis row (RH + 2 signal cards) span ~30% of viewport before the tab strip renders. The first thing a 1700px viewport sees is the relationship.
  • G2Sub-tabs are lenses, not departments. Same 5 tabs as mobile. RH sentence at top unifies what each will contain. Tab strip is the same quiet style — no badges, no color, no department-app feel.
  • G3Satisfaction conditional. Hernandez has no CS signals (no review, no warranty, no sentiment shift). Tab does not appear. 5 lenses carry the page.

Desktop density choices

  • Synthesis row 3-column · RH summary takes the largest cell, two highest-value signals to the right as cards (CO pending · last touch). On mobile these were rows in a stack — desktop puts them side-by-side without losing the read-order.
  • Project card state grid 2×2 · framing status · CO pending · base contract · cabinetry hold. Mobile was a single column.
  • Continuity strip inline · Field / Schedule / Money pulled into the project card body — same continuity F-PR2 carries, surfaced one level earlier at desktop density.
  • Right rail · CO inline preview · the highest-impact action (approve CO-002) is one click away from the client lens, not buried behind "Open project."
  • Recent touches timeline · 4 most recent · icons typed by channel (field / email / call / artifact). Right-rail placement keeps it scannable without dominating.
  • Past projects collapsed · single Powder room refresh card · opacity-dimmed to signal closed · summary line + view CTA only
