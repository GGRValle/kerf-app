F-CL2 · Mobile Client Detail · relationship lens, not workstation

Per-client lens. Reached by tapping a client row from F-CL1. The surface where Tier 2 doctrine is most at risk — the natural temptation is to build a CRM workstation with a tab strip dominating the page, contact metadata blocks everywhere, separate mini-dashboards inside each tab, and a Satisfaction tab that sits ceremonially empty. This surface refuses every one of those moves. Lead with the client state. Tabs are secondary lenses, not departments. Per feedback_business_brain_not_file_cabinet.md + the visual review preface: "A surface should do one primary cognitive job."

Three hard guardrails applied (Christian, 2026-05-17 PM late): (1) Lead with client state, not tab strip — the first thing visible is who this client is and what's happening with them; tabs render below as secondary lenses. (2) Sub-tabs are lenses, not departments — the RH summary at top unifies them; the tab strip is visually quiet. (3) Satisfaction conditional and quiet — surfaces only when Client Success agent has signals (referral pending · recent review · warranty item · sentiment shift). For Hernandez (new active client, no CS signals), Satisfaction does NOT appear. Base 5 tabs carry the page.

Three guardrails applied

  • G1Lead with client state, not tab strip. The first 60% of the surface is client identity + RH summary + 3 signals. Tab strip renders below as secondary. The visual weight of the page lands on the relationship, not on the tabs.
  • G2Sub-tabs are lenses, not departments. The RH summary at top says "pantry conversion in flight · CO-002 awaiting your approval · field walk captured today." That single sentence unifies what each tab will eventually contain (Projects · Comms · Notes · Files · Relationships). The tabs are different ways to look at the same relationship, not five separate modules.
  • G3Satisfaction conditional and quiet. Hernandez has no CS signals yet (new active client, no reviews/referrals/warranty events). Therefore the Satisfaction tab DOES NOT appear. Base 5 tabs carry the page. If Hernandez later generates a review or signs a referral, Satisfaction would appear as a 6th tab — but only then.

Top section anatomy

  • Client name · 20px · 700 weight · primary visual anchor
  • Phase posture · "Active client · address · since [date]" · one line of context
  • RH summary · one sentence · synthesizes state across what the tabs will show · amber accent on the attention item
  • 3 high-value signals · active jobs · last touch · current attention item · each on its own row with icon + headline + sub-detail. Left-border amber on the attention-needed signal.

Tab strip — visually secondary

  • Renders below the signals, with subtle styling — small text, light count chips, single-pixel underline on active tab
  • Tabs don't dominate. The active tab indicator is the only amber element in the strip.
  • Horizontal scroll if needed (5 tabs fit at this width; Satisfaction would push to 6 and trigger scroll)
  • Default selected: Projects — because projects are the business consequence center per Christian's call

Projects tab default content

  • Project card · name · phase tag (Active / Closing / Archive) · address · 2-3 state rows · CTA to open project
  • State rows use 3-color dots: green (on track) · amber (pending action) · neutral (informational)
  • Money rendered in dollars (storage-as-cents per CLAUDE.md)
  • + New project for [Client] · dashed-border affordance · routes to F-S1 → Start estimate/proposal pre-bound to this client

Per-client lens model (each tab as a perspective)

  • Projects (default): current + past projects for this client, sorted active-first
  • Comms: SMS · email · portal threads · field-walk transcript pointers — chronological under the relationship
  • Notes: RH-aware notes + voice memos about this client (tribal knowledge)
  • Files: client-specific files (intake forms, signed agreements, mood boards) — under the client, not floating
  • Relationships: linked designers · architects · realtors · referral sources — the relational substrate
  • Satisfaction (conditional): sentiment · reviews · referrals · warranty items · appears ONLY when CS agent has signals

What this surface refuses to be

  • A contact record with every field visible (no metadata block · no birthday · no address-line-2 spelled out)
  • A communications archive home (Comms tab carries that, secondary)
  • A note bucket
  • A referral admin console
  • A tabbed filing cabinet
  • A workstation with 5 mini-dashboards stitched together
  • A "click to edit 47 fields" intake form
  • A Salesforce-style profile page

Doctrine bound

  • d-1One primary cognitive job — see this relationship's state at a glance, drill where attention is needed
  • d-2Smart summary leads — RH synthesis line + 3 signals · before any tabs
  • d-33-color discipline preserved — amber pending · green on-track · red drift · neutral informational
  • d-4Phase as text tag, not status color — Active / Lead / Repeat shown as small tag, distinct from state colors
  • d-5Trust surfaces stay exception-tier — no transcript or audit nav on this page; reached only via project-detail drill
  • d-6Telemetry stat-grids forbidden — 3 signal rows are action-driving (count + context + detail), not telemetry
  • canonRight Hand framing once at top · no agent names elsewhere
  • canonPer kerf_client_section_canonical.md · Clients first-class, upstream of Projects

What this surface proves The relationship-lens framing holds even when the temptation is highest. A tab strip exists because the IA v3 spec requires it; the design refuses to let it dominate. Lead with client state. Tabs are lenses underneath. The RH summary sentence carries what each tab will eventually contain — so even before the operator picks a tab, they already know the relationship's state. That's the relationship lens working.
What this surface refuses to be Not a Salesforce-style profile page. Not a Houzz Pro contact card with every field visible. Not five mini-dashboards stitched together. Not a tabbed filing cabinet. F-CL2 is a relationship synthesis with tab-shaped drill affordances underneath. Satisfaction stays invisible until CS signals exist — no ceremonial empty tab.
