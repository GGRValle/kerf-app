4-question dispatch

  • Macro / comparison / detail? · comparison (peer subs across roster)
  • List, grid, both? · BOTH · List default (attention-first) · Grid for macro scan
  • Cognitive job? · who do I have · who's reliable · who's at risk · who's working what
  • What stays hidden? · individual job assignments · payment history · communication threads → drill into Sub Detail

Doctrine applied

  • d-1One primary cognitive job — see roster state, drill where risk lives
  • d-2Smart summary leads — RH synthesis line at top, not stat grid
  • d-3Sort by attention-needed · no-show / liability lapse → insurance expiring → bid quiet → calm → closed
  • d-4Trade as text tag · status colors reserved for state (red = liability/no-show · amber = COI/bid quiet · green = stellar)
  • d-5Red is chip-tier, not row-tier · Westside Tile carries red chip, row stays neutral
  • d-6No agent names · "insurance flagged" not "Field Hand flagged"
  • d-7Translate canon vocab · "your subs" not "the sub cohort"

Row anatomy

  • Company name · primary text · scannable
  • Trade tag · text-only tag (Tile · Drywall · Plumbing · Electrical · Glass · HVAC · Cabinetry · Demo · Roofing · Stone)
  • Meta · contact name · 1099 status · jobs YTD
  • State chip · only when action implied (red no-show · amber insurance/bid quiet)
  • State one-liner · what's happening with this sub right now
  • Left-border accent · amber for pending action · neutral for calm
  • Arrow · drill-in to F-SBD Sub Detail

Section order (locked)

  • Attention · subs with active flags (no-show · insurance · bid quiet)
  • Active subs · everyone in good standing, sorted by current job rhythm
  • Closed / inactive · muted · for reference and re-engagement

Subs vs Crew separation

  • 1099 subcontractors only · NOT W2 employees (those live in Crew · sibling sub-item)
  • COI tracking is the gating compliance signal (W-9 + COI required to work)
  • Bid-quiet and no-show flags are sub-specific reliability signals
  • Different cadence than crew · less daily-present, more job-by-job

Routes

  • Back arrow → Team & Ops landing (or Home)
  • Add → F-S1 Start sheet → Add sub workflow
  • Sub row tap → F-SBD Sub Detail (jobs · COI · payments · communication)
  • Westside Tile chip tap → drift detail / no-show audit
  • Hector's Drywall amber chip tap → COI upload workflow
  • 🎤 Speak (long-hold) → RH conversation pre-bound to subs roster context
  • Bottom bar tabs → respective destinations · no tab active (sub-view)

  • Not a Rolodex with 47 contact fields
  • Not a job-assignment kanban
  • Not a multi-tab vendor hub
  • Not a filter-and-sort grid with checkbox bulk operations

F-SB1 Grid · Spec

Big-picture canon applied

  • Comparison view · every sub visible at once
  • 2-column mobile · 12 tiles · no attention compression
  • Tile content disciplined: name + trade + 1099 + state chip + rhythm line · scan not detail
  • State chip palette: red (no-show / liability) · amber (COI / bid quiet) · cyan (critical path / closing) · green (stellar)
  • Map view (V1.5+) would be third toggle option · deferred (geo-routed sub dispatch)

Why Grid matters for Subs

  • Owner needs to see the whole bench when deciding who to call · "who's ready right now"
  • Stellar-vs-flagged comparison is the core question on bid-out · grid puts both on screen together
  • Trade-by-trade visual scan: 3 tile subs, 2 plumbing, 2 electrical · roster shape is the answer
What this surface enforces Roster state, not vendor database. The first sentence tells the owner what's happening with the bench in plain English. The list sorts by attention-needed so the first row is always the sub that matters most this minute. Trade tags are small text labels; status colors are reserved for state. The 80% case (see bench state, drill where risk lives) handled by default. Subs is distinct from Crew — different obligations, different cadence, different roster.
What this surface refuses to be Not a vendor database. Not a Rolodex with 47 fields per card. Not a procurement system with PO tracking. Not a kanban with "Available / Booked / Working / Done" columns. F-SB1 is the bench synthesis with drill affordances underneath. Same lens-not-console discipline as F-PR1, scoped to the 1099 roster.
