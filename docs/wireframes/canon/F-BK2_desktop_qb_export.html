F-BK2 · Spec

4-question rule

  • Macro / comparison / detail? Detail · action surface (export pipeline)
  • List / grid / both? Neither · Grid Doctrine refused on detail/action surfaces (D-046)
  • Cognitive job? Generate IIF · confirm validation passes · confirm operator before export · audit-trail the run
  • Hidden until drill? Per-transaction history lives in F-BK1a/b recon · IIF file structure invisible (operator just gets the download)

Density · 17-entry preview · 184 total

  • Preview sectioned by type: AR · 6 + AP · 7 + Journal · 4 = 17 visible (preview discipline)
  • Full export: 184 entries · $48,200 total · virtualized at scale
  • History table: 6 prior runs (rolling window)
  • Service-trade stress (50+): preview virtualizes · validation runs full set · IIF size scales linearly

Money model · QB pipeline

  • Display: dollars · Store: integer cents on every amount
  • Lifecycle: staged → validated → exported → (V1.5) synced_back
  • V1 export-only · QB changes after export do NOT sync back · operator handles dedup if re-exporting same range
  • Validation triad: orphan transactions · duplicate refs · GL balance ties out (all 3 must pass before Export IIF enables)

Fillable function (memory canon)

  • Fill-strip surfaces who ran the export · Owner / Delegate / External MCP
  • History table records operator + role per run (audit trail)
  • Per feedback_bookkeeping_is_a_fillable_function.md · QB export is a Bookkeeping sub-task, inherits the fillable canon

Sidebar selection

  • Team & Ops active · Bookkeeping › QB export sub-item active
  • Breadcrumb: Home › Team & Ops › Bookkeeping › QB export

Print affordance (Lane 7B)

  • Top-right bc-bar carries Print preview · prints the 17-entry sectioned preview table + validation triad state
  • IIF Export is the surface's primary purpose · separate Export ▾ not added (would duplicate the surface)
  • Audit trail covers both IIF download and Print event
What this surface enforcesValidate before export · explicit operator action to download. V1 export-only disclosure surfaced first. Fill-strip and history table record the role that ran the job. Three-check validation (orphans · duplicates · balance) must pass before the Download button enables.
What this surface refuses to beNot a grid surface (Grid Doctrine refused on detail/action). Not bidirectional sync (V1.5 feature). Not a per-transaction inspector (drill back to F-BK1a/b for line items). Not auto-export — every run requires operator approval + role assignment.
