F-PR1 · Mobile Projects list · portfolio state, not project database

Tier 2 Projects entry. Reached by tapping Projects in the Owner default bar. Same lens-not-console discipline as F-CL1. Per the doctrine preface: "A surface should do one primary cognitive job." F-PR1's primary cognitive job is "see the state of my active project portfolio and drill where work is happening." Not a project database. Not a Gantt chart. Not a kanban with 47 fields per card. Right Hand summary leads, rows sort by attention-needed (drift first, then pending decisions, then on-track), phase shown as plain text tag, current state one-liner color-coded only when action implied.

Doctrine applied (same as F-CL1, scoped to projects)

  • d-1One primary cognitive job — see project portfolio state, drill where work is happening
  • d-2Smart summary leads — RH synthesis line at top, not stat grid
  • d-3Sort by attention-needed, not alphabetical, not recently-updated · drift → pending decisions → on-track
  • d-4Phase as text tag, not status color · status colors stay reserved for state (amber pending · red drift · cyan closing)
  • d-5Portfolio summary at bottom, not header · reference content doesn't lead the page
  • d-6No filter bar · default sort handles 80% · long-press for explicit filter
  • d-7No transcript/audit nav · trust surfaces stay exception-tier

Row anatomy

  • Project name · primary text · scannable
  • Phase tag · small text-only tag (Active / Bid / Lead / Closing / Archive) · distinct from state colors
  • Meta · client name · address · last touch · one line
  • State one-liner · what's happening with this project right now · color-coded when action implied
  • Left-border accent · red for drift · amber for pending action · cyan for closing · neutral for on-track
  • Arrow · drill-in to F-PR2 Project Detail

Section order (locked · revised 2026-05-18 PM late+ per D-046 scale lens)

  • Active · projects in flight (sorted within: drift → pending → on-track) · default ~6 jobs for GC archetype
  • Closing · finishing this week or next · distinct from Closeout (still in delivery, not yet punch list)
  • Bid · in proposal/bid stage · feeds from Sales pipeline · shown here so Owner sees the full pipeline-to-handoff arc in one place (revised from prior "not shown for Owner" rule)
  • Closeout · punch list + retainer release (cyan accent)
  • Warranty · post-handoff callbacks (low-frequency · lives here so they don't fall out of sight)
  • Archive · long-press → archived projects (not on default view)

Scale lens · 15-20 baseline · 50+ stress test

  • GC archetype baseline: 15-20 active projects across the 5 sections (this surface's default sample)
  • Compression rule: when a section has >5 attention items, show the top 5 + "+ N on-track" rollup (Active section here demonstrates with 3 attention + 3 rolled up)
  • Service-trade stress test (50+ work items): compression rolls up aggressively (top 5 attention + "+ 42 on-track" pattern) · same row anatomy · Grid view is the "show me everything" override · long-press section header → expanded view of the rolled-up jobs
  • Filter/sort affordances: long-press any section header for explicit filter (deferred surface) · default sort handles 80% of operator scan-and-drill behavior

Portfolio footer

  • Aggregate financial signals (Total contracted · AR state · lookahead) · reference, not telemetry
  • AR-late shows amber when late (action-driving) · neutral otherwise
  • Lookahead summary points to Schedule destination (when built)
  • Margin NOT shown at portfolio level on this surface · operator-private, lives in Money destination

Routes

  • Back arrow → F-A1 Owner Home
  • New → F-S1 Start sheet → Start estimate/proposal (intent-first)
  • Project row tap → F-PR2 Project Detail
  • Portfolio AR-late tap → Money > AR (Tier 3 surface — deferred)
  • Portfolio lookahead tap → Schedule destination (Tier 3 surface — deferred)
  • 🎤 Speak (long-hold) → RH conversation pre-bound to project portfolio context
  • Bottom bar tabs → respective destinations · Projects stays active

What this surface refuses to be

  • Not a project database
  • Not a Gantt chart
  • Not a kanban board with status columns
  • Not a multi-tab project hub
  • Not Houzz Pro's project tiles with 47 fields per card
  • Not a filter-and-sort dashboard with multi-select chips

F-PR1 Grid · Spec

Big-picture canon applied

  • Comparison view · every active job visible at once
  • 2-column mobile · 10 tiles fit one screen scroll · no attention compression
  • Tile content disciplined: name + phase + state chip + rhythm line · scan not detail
  • Map view (V1.5+) would be third toggle option · deferred
What this surface enforces Portfolio state, not project database. The first sentence tells the operator what's happening with their portfolio in plain English. The list sorts by attention-needed so the first row is always the one that matters most this minute. Phase tags are small text labels; status colors are reserved for state. The 80% case (see state, drill where work is) handled by default.
What this surface refuses to be Not a project database. Not a Gantt chart. Not a kanban with multi-status columns. Not a project hub with 12 quick filters and a "Create Project" wizard at the top. F-PR1 is the portfolio synthesis with drill affordances underneath. Same lens-not-console discipline as F-CL1, scoped to projects.
