F-AD1 · Mobile Admin landing · work queue, not bookkeeping ledger

Tier 2 Admin entry inside the Team & Ops domain. Reached by tapping More › Team & Ops › Admin. Per the doctrine preface: "A surface should do one primary cognitive job." F-AD1's primary cognitive job is "see what admin needs my hand this week and what can be delegated." Not a transaction ledger. Not a full document library. Not an HR roster. Right Hand summary leads with two time-sensitive items called out by name; rows section by Time-sensitive / Needs your hand / Delegate-ready; List default attention-first, Grid for macro scan. The admin role here is fillable — Owner, agent, external bookkeeper MCP, or future Admin/Ops role-root all clear items from this same queue.

Owner tapped Admin. First thing visible: synthesis line — "7 admin items need clearing. 2 time-sensitive (CSLB renewal 14d · Saul COI 9d). 3 can be delegated." Below: rows sectioned by Time-sensitive (3 amber/red), Needs your hand (4 amber), Delegate-ready (3 muted with blue chip). Each row carries name + plain-English context + state one-liner. No agent names. No ledger views. Drill into any row for the specific item.

Build position

Grid doctrine applied

  • canonDomain list = comparison · admin landing IS a comparison surface (peer admin tasks)
  • canonList + Grid both required · List default attention-first · Grid for macro scan
  • d-1One primary cognitive job — what admin needs my hand this week, what can be delegated
  • d-2Smart summary leads — RH names the 2 time-sensitive items by name + count of delegate-ready
  • d-3Sort by attention — Time-sensitive → Needs your hand → Delegate-ready (sections, not just within)
  • d-4Phase as text tag · status colors reserved for state (amber pending · red critical · blue delegate · cyan closing)
  • d-5Red is chip-tier only — never full-row red border (per 2026-05-18 lock)
  • d-6No agent names — "renewal due" not "Compliance Agent flagged"
  • d-7Translate canon vocab — "your admin work" not "admin queue cohort"

Row anatomy

  • Item name · primary text · plain English (CSLB license renewal, not COMPLIANCE-001)
  • Phase tag · small text-only tag (License / Insurance / Payroll / Bookkeeping / Contracts / Permits / AP / Compliance / Vehicle)
  • Meta · who / what / amount / context · one line
  • Days-left chip · only on time-sensitive section · red <15d · amber <30d
  • State one-liner · what action this needs · color-coded when action implied
  • Left-border accent · amber for needs-your-hand · neutral border for delegate-ready · no full-row red
  • Arrow · drill-in to admin item detail (F-AD3+ deferred)

Section order (locked)

  • Time-sensitive · <30 day clocks (license · COI · vehicle reg · permits with deadlines)
  • Needs your hand · approval / signature / review items · owner-only authority
  • Delegate-ready · routine work safe to hand off · muted to signal lower attention
  • Cleared · long-press / Actions → show recently cleared (not default view)

Mini-tabs (filter scope)

  • All · default · shows the full queue across all admin types
  • Mine · only items where the operator is the action-holder
  • Delegate-ready · items safe to hand to bookkeeper MCP / external admin
  • Time-sensitive · only items with <30 day clocks

Fillable role

  • Same queue surfaces for Owner / external bookkeeper MCP / future Admin/Ops role-root
  • Role projection filters which items appear and what "Mine" means
  • Delegate-ready chips name the role, not a person (Bookkeeper · not "Sarah")

Routes

  • Back arrow → More menu (or F-A1 Owner Home)
  • New → New admin item sheet (vendor add · permit kick-off · etc.) · F-AD-NEW deferred
  • Row tap → admin item detail (F-AD3+ deferred · varies by phase)
  • Delegate-ready chip tap → assign-to-bookkeeper sheet
  • 🎤 Speak (long-hold) → RH conversation pre-bound to admin queue context
  • Bottom bar → respective destinations · More stays active (Admin is under Team & Ops)

What this surface refuses to be

  • Not a bookkeeping ledger (no transaction list, no chart of accounts)
  • Not a full document library (no doc browser, no folder tree)
  • Not an HR roster (Crew lives in its own sibling under Team & Ops)
  • Not a QuickBooks console (sync state surfaces only when sync needs action)
  • Not a compliance dashboard with charts and audit-trail widgets
  • Not a Houzz Pro / Buildertrend admin tab with 40 navigation items

F-AD1 Grid · Spec

Macro-scan canon applied

  • Comparison view · every admin item visible at once
  • 2-column mobile · 10 tiles fit one screen scroll · no attention compression
  • Tile content disciplined: name + phase + chip + rhythm · scan not detail
  • Delegate-ready tiles use neutral left border + dimmed opacity to read as lower-attention without going red/amber
  • Drill into any tile → admin item detail (F-AD3+ deferred)

What Grid view refuses to be

  • Not a kanban (no drag between status columns)
  • Not a dashboard with micro-metrics per tile
  • Not a Houzz Pro admin grid with 40 fields per card
  • Scan-first · disciplined per the big-picture canon
What this surface enforces Work queue, not bookkeeping ledger. The first sentence tells the operator how many admin items need clearing and names the two time-sensitive ones. The list sorts by attention so the row that matters most this minute is always visible first. The Delegate-ready section signals "hand this off" without burying it — the muted treatment reads as lower-attention without going red/amber. Admin is fillable: same surface, different role projections, same cleared-items output.
What this surface refuses to be Not a bookkeeping ledger. Not a document library. Not an HR roster. Not a QuickBooks console. Not a compliance dashboard. F-AD1 is the admin work queue — items needing the operator's hand — with drill affordances underneath. Individual transaction ledgers and full document libraries live one level deeper or in their own sibling destinations under Team & Ops.
