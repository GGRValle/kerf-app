F-CL3 · Spec

Build position

Tier 2 · Clients desktop
✓F-CL1 · Mobile Clients list

What desktop adds vs F-CL1 mobile

  • Left sidebar always visible · all 9 domains accessible · Clients active
  • Window chrome + breadcrumb + global search
  • 2-column main · list left (wider, more state inline) + portfolio rail right
  • Portfolio rail surfaces: RH synthesis · at-risk callout · pipeline funnel · referral-path card
  • Rows show state inline rather than below name (more horizontal real estate)
  • Add client as breadcrumb-bar action affordance (not bottom-right FAB on mobile · routes to F-S1 Start client intake)

What stays from F-CL1

  • Attention sort (not alphabetical)
  • Three sections: Active / Leads / Repeat
  • Phase shown as plain text, not status color
  • Status colors reserved for state (amber pending · red drift · green happy/referral)
  • Left-border accent on rows needing attention
  • No filter bar default
  • No tabs · single scannable list with sections

What this surface refuses

  • A grid of contact cards with avatars and metadata
  • A multi-tab CRM workstation (Active / Past / Leads as tabs)
  • A filter sidebar with 15 facets
  • A bulk-action toolbar
  • An alphabetical default sort

Doctrine bound

  • canonSame lens-not-console discipline as mobile
  • canonSmart summary leads (RH card)
  • canon3-color discipline preserved
  • canonTrust surfaces stay exception-tier (no transcript or audit nav here)
  • canonLight + dark theme baked in
