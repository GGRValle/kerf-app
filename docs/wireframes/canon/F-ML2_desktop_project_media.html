F-ML2 · Spec

Build position

Lane 6 · Project Media Library
✓F-ML1 · Mobile Project Media
2F-ML2 · Desktop Project Media (this)
3F-PR2 / F-PR4 patch · Media sub-tab

4-question rule (F-FD1 pattern)

  • Macro / comparison / detail? Detail/browse — per-project media lens at desktop density, not portfolio comparison.
  • List / grid / both? Thumbnail grid only · NO List/Grid toggle (detail surface · grid is primary · same canon as F-ML1).
  • Cognitive job? See what was captured on this job at desktop scale · filter to photos/videos/dailies · jump to fullscreen viewer or daily log.
  • Hidden until drill? Full history (lazy-load · virtualizes at 500+) · fullscreen viewer · daily log body (F-E1 anchor).

IA decision

Per-project drill only. Media is not a top-level sidebar destination. Route: Projects → [project] → Media sub-tab (or voice deep-link). Breadcrumb carries wayfinding.

Desktop density

  • 6-column thumbnail grid · 30 items default visible (vs 3-col · 20–30 on mobile)
  • Rail-led synthesis · RH summary + phase scrubber + rollup moved to right rail; grid dominates main column
  • Same filter chips + sort as F-ML1 · All · Photos · Videos · Daily logs · By date · By phase · By capturer · Sort: Most recent · Oldest first · By phase
  • Lazy load / pagination · initial fetch 30 · infinite scroll loads next 30 · at 500+ switch to virtualized window (only DOM nodes in viewport)

Voice retrieval routing

  • "Wegrzyn kitchen photos" → F-ML2 · project=Wegrzyn · filter=Photos
  • "Wegrzyn videos" → F-ML2 · project=Wegrzyn · filter=Videos
  • "Most recent dailies from Hernandez" → F-ML1/ML2 · project=Hernandez · filter=Daily logs · sort=Most recent
  • "Show me the pictures" (project in context) → F-ML2 · filter=Photos · sort=Most recent
  • "Right Hand, label this framing day 3" → voice caption on selected thumb (viewer open)

Doctrine

  • d-1One cognitive job — browse all project captures without scrolling daily logs
  • d-2RH summary leads — rail card opens synthesis before grid scan
  • canonRed chip-tier only · no full-tile red borders on flagged photos
  • canonRight Hand named once · no agent names elsewhere
  • canonSingle source of truth · same asset in Daily Log + Media Library

Routes

  • Thumb tap → Frame 2 viewer (or inline split at desktop width)
  • Open daily log → daily log for that capture date
  • Breadcrumb Wegrzyn → F-PR4 Project Detail
  • Phase scrubber drag → filter grid to date range for that phase

F-ML2 Frame 2 · Spec

Alternate states (either/or)

  • Daily logs filter active — chip highlighted · grid shows log-linked items · count reflects log total
  • Viewer panel at desktop width — split layout below grid · prev/next nav · metadata rail · no fullscreen takeover on desktop

Viewer actions

  • Open daily log → parent log for capture date
  • Share → standard share sheet (no auto-send)
  • Label with voice → caption edit via Right Hand · writes to metadata

Lazy load note

At hundreds of items: intersection observer on sentinel row triggers next page fetch. Server returns cursor + next 30 IDs. Client merges into virtualized grid — no full reload.

What this surface proves A real kitchen remodel accumulates hundreds of photos over 11 weeks. Kerf does not bury them inside daily log scroll. Flat browse + phase scrubber lets the owner or Right Hand land on "all kitchen photos" in one gesture — voice or click. Same asset references Daily Log; no duplicate storage.
What this surface refuses to be Not CompanyCam's folder tree. Not a top-level "Media" nav item. Not a file cabinet with arbitrary albums. Not agent-named retrieval. F-ML2 is F-ML1 with desktop breathing room — 6 columns, rail synthesis, inline viewer.
