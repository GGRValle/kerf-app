F-E1 · Mobile Field Capture · voice + photo + Daily Log anchor

Rebuild of F-33 with v2 chrome conventions and tightened Daily Log canon. The entry point of the field-capture loop — operator opens the app on a jobsite, captures voice + photos, picks what Kerf should do with the result. Every capture writes to the per-(tenant × project × day) Daily Log regardless of draft intent, per kerf_daily_log_canon.md. Three states: pre-capture setup (assign client/project/location, start) → active capture (live transcript, accumulating items) → pre-flight + submit (review, pick intent, send). Apple's speech-to-text handles transcription; transcript follows the three-part canon (original immutable + edits overlay + current rendered) per kerf_transcript_canon.md. Plain English throughout; no agent names in operator copy.

Frame · F-E1 · Mobile Field Capture · 3 states · Daniel · field lead at Wegrzyn jobsite

Three-state grid. State 1 starts with assignment (the capture has to know which Daily Log it lands in before anything else); the Daily Log anchor is canon-explicit on the surface so the operator sees where this is going. State 2 is the active recording session with live transcript streaming from Apple's speech-to-text; captured items accumulate as a list. State 3 is the pre-flight gate — operator reviews the transcript, picks what Kerf should do with the capture (draft a change-order · draft an estimate update · save to Daily Log only), and submits. The third option is the trust-discipline path — when the operator is unsure, no draft is forced.

What changed from F-33 → F-E1

  • All emoji icons replaced with silhouette SVGs (mic, camera, pin, document, etc.) per v2 chrome conventions
  • Daily Log anchor banner added on State 1 — canon-explicit visible to operator that every capture writes to the project's Daily Log per kerf_daily_log_canon.md
  • Transcript canon hint added on State 3 — operator sees that edits overlay the original (original immutable + edits as overlay) per kerf_transcript_canon.md
  • Agent-name leaks removed from operator copy: "Estimator drafts faster" → "Kerf drafts faster"; "iOS speech-to-text" naming removed from operator-visible label (kept in spec); "the Estimator runs the workflow" → "a draft is prepared"
  • AI disclosure §13 added on State 3 — *"AI-assisted by Kerf Right Hand · Review before approval"*
  • Color discipline: record button uses field-green (capture is field-side work; matches F-C1 accent) instead of red (which is reserved for drift/blocker per the 3-color rule)
  • Bilingual asymmetry preserved: single-language UI per tenant locale; agent identity bilingual on the Speak button (already canon)

Daily Log canon visible at the surface

  • State 1 shows the Daily Log anchor banner: "Writes to Wegrzyn · today's Daily Log."
  • The operator sees this BEFORE recording, so the destination of the capture is never a mystery
  • Per kerf_daily_log_canon.md: per-(tenant × project × day) Daily Log. Captures don't float as orphan uploads.
  • Even "Save to Daily Log only" is the explicit third option — when an operator is unsure what the capture means, no draft is forced

Transcript canon visible at the surface

  • State 3 shows the transcript canon hint: "Edits overlay the original. Your edits are saved as changes on top of what was recorded — the original transcript stays intact for audit."
  • Per kerf_transcript_canon.md: three-part schema — `transcript_original` (immutable Apple verbatim) + `transcript_edits` (operator overlay events) + `transcript_current` (rendered working version)
  • Operator never has the "did I overwrite my original?" anxiety — the canon makes it impossible

Three states · canon-anchored

  • State 1 · Pre-capture. Assignment must be set first (client + project + location). Daily Log anchor explicit. Record button is the dominant action (field-green); photo and note as secondaries. Submit is disabled until at least one capture exists.
  • State 2 · Active capture. Record-bar at top with timer and stop affordance. Live transcript card shows speech-to-text streaming (current word in field-green with cursor). Captured items list accumulates voice clips + photos + notes; each item has timestamp + remove affordance. Done button promotes to State 3.
  • State 3 · Pre-flight + submit. Capture summary card with stats. Draft-intent picker (3 options: CO · estimate · Daily Log only) — the critical decision before AI does anything. Final transcript visible with edit affordance + canon hint. Photos in compressed grid. Submit indicates expected wait (~30s for CO/estimate paths).

The draft-intent picker is the critical gate

Operator's choice in State 3 routes the capture through different downstream paths:

  • Draft a change-order → CO workflow runs · a CO draft is prepared (priced, routed for owner approval) · operator lands on F-G1 Draft Review with CO-shaped artifact
  • Draft an estimate update → estimate workflow runs · line additions / edits proposed · operator lands on F-G1 Draft Review with estimate-shaped artifact
  • Save to Daily Log only → no draft workflow · capture files as field evidence on the project · triage later if scope changes

Default selection is operator-driven. The system may suggest based on voice content, but the operator confirms. Trust-discipline: when unsure, no draft is forced.

Apple owns capture · Kerf wraps

Same boundary as F-32 with RoomPlan and F-S1 takeoff routing. Apple's speech-to-text framework handles voice → text streaming. Kerf wraps:

  • Voice clip storage (reference audio retained for re-transcription)
  • Transcript editing per canon (original immutable + edits overlay)
  • Background transcription (long memos process while operator continues)
  • Per kerf_lidar_precision_limits_and_apple_wrap.md — same Apple-wrap discipline applied here for speech

  • Back from State 3 → returns to State 2 with session intact

What this surface enforces Capture once · consequences propagate. Every capture writes to the Daily Log regardless of draft intent. The draft-intent picker is the explicit gate where the operator chooses what Kerf does with the capture — not a silent inference. Trust-first: when unsure, no draft is forced (Save to Daily Log only is a first-class option).
What this surface refuses to be Not a record button on its own. Not a "tap to upload" file picker. Not an Estimator launcher. Not a chronological feed of past captures. F-E1 is the entry point where field reality lands in a Daily Log and the operator picks the consequence path.
