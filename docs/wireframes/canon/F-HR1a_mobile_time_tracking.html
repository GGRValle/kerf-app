F-HR1a List · Spec

4-question rule

  • 1Macro / comparison / detail? Comparison — peer time entries this week
  • 2List / grid / both? Both — List default, Grid for macro scan
  • 3Cognitive job? What hours need approval? Any anomalies?
  • 4Hidden until drill? Per-employee history, payroll export path

Lifecycle states

  • Time entry: submitted · approved · exported (last week)
  • Flags: chip-tier only — never full-row red

Export / Print affordances (Lane 7B)

  • Actions strip below RH summary · Print + Export ▾ weekly summary · PDF · CSV · XLSX
  • Existing Export to QB/payroll provider (rail CTA) stays as primary payroll egress · Lane 7B Print adds weekly-summary printable for archive
  • Audit trail on every Export event: operator + week + format · payroll lifecycle untouched

F-HR1a Grid · Spec

  • 2-col tiles · gt-name = employee · gt-phase = week total · gt-chip = flag state · gt-rhythm = site
What this surface enforcesApproval queue, not timesheet editor. Right Hand summarizes the week; anomalies surface as chips. Standard entries compress to batch approve. Professional HR tone — flags describe facts, not judgment.
What this surface refuses to beNot a full payroll system. Not per-punch editing at list density. Not employee performance commentary. Drill to crew/employee detail for history.
