F-B1c · Mobile Decision Card · Client preview · what the homeowner sees

What the client receives. Reached via the "Preview client view" button on F-B1 (pending) or F-B1b (edit). PDF-style rendering on light paper inside the dark phone chrome — the operator can see exactly what's going to the homeowner. Itemization level controls below the document let the operator pick: Detailed (every line + labor/material breakdown + markup visible), Summary (category groupings with totals, no internal breakdown — the default for residential homeowners), or Lump sum (scope description + single total, used when relationship trust is high or when itemization would distract). Toggle to show or hide margin %. Client-facing AI disclosure per kerf_ai_disclosure_pattern.md: no "AI-assisted by Right Hand" branding on the document — contractor is the accountable signer.

Entry points

  • F-B1 (pending) · "Preview client view" link
  • F-B1b (edit) · "Preview client view" secondary action above Save
  • Project Detail > Scope > CO-002 · context menu "Preview"

Itemization levels

  • Detailed · every line + labor/material/sub breakdown · margin can be shown or hidden · used with designers, sophisticated clients, commercial, public bid
  • Summary (default) · category groupings with totals · no internal breakdown · used with most residential homeowners
  • Lump sum · scope description + single total · used when relationship trust is high or itemization distracts from scope

Show margin % toggle

  • Off by default · most residential clients shouldn't see contractor margin
  • On for designer relationships (often have margin-share or transparency contracts)
  • On for commercial / public bid (often required by contract type)
  • Saved per-client preference once set

What the document shows (Summary level here)

  • Letterhead · contractor's brand · license · contact
  • Doc title · "Change Order · Pantry Scope Expansion · CO-002 · modifies EST-014"
  • Project info · Date · Client · Property · Phase
  • Scope of Work · plain-English intro + bullet list of work
  • Investment Summary · category rows + materials buffer + allowances · single total
  • Acceptance · authorization paragraph + dual signature lines (client + contractor)
  • Footer · page · CO id · date

Client-facing AI disclosure (§13)

  • CLNO "AI-assisted by Right Hand" branding on the document
  • CLNO Kerf branding visible on the artifact
  • CLContractor is the signer · contractor-of-record accountability
  • CLFooter may optionally include tenant-set "Generated with Kerf" line (off by default)
  • This is the canonical client-facing artifact rule per kerf_ai_disclosure_pattern.md

What changes between Detailed / Summary / Lump sum

  • Detailed · investment table expands to ~14 rows · per scope line: labor hours · material cost · sub cost · markup if toggle on · line total
  • Summary (shown) · ~7 rows · category groupings + materials buffer + allowances · single total
  • Lump sum · investment block collapses to 2 rows · "Pantry scope expansion · $14,820" with a 1-sentence inclusion summary
  • Scope of Work section unchanged across levels (always shown in plain English)
  • Acceptance section unchanged across levels

Operator UX

  • Tap itemization segment → document re-renders inline (no page reload)
  • Toggle margin → margin column appears/disappears in Detailed level
  • Done returns to entry point (F-B1 or F-B1b)
  • Back to edit returns to F-B1b regardless of entry
  • Share icon (top-right) opens send sheet (V18 send gate — NOT auto-send)

Per-client preference memory

  • Itemization level + margin visibility saves per-client on Done
  • Next CO/proposal for same client defaults to the saved preference
  • Operator can override per-document at any time
  • Stored in tenant memory · not visible to client

Level A
Detailed

Every line broken out · labor hours · material cost · sub cost · line total. Margin column optional via toggle.

When to use: designers, sophisticated repeat clients, commercial GCs, public bid responses.
Level B · default
Summary

Category groupings (framing, cabinetry, electrical, etc.) with totals only. No internal breakdown, no margin.

When to use: most residential homeowners. Clear enough to trust, simple enough not to invite line-by-line negotiation.
Level C
Lump sum

Scope description + single total. One-paragraph what's-included summary. No itemized rows.

When to use: high-trust repeat clients, design-build with allowance contracts, when itemization distracts from scope.
Why itemization control matters Different clients warrant different transparency. A designer wants full labor/material breakdown for value engineering; a homeowner gets confused by it and starts negotiating individual lines; a commercial GC contractually requires it. The same draft, three different client-facing presentations — selectable on this surface, saveable per-client. The substrate doesn't change · the projection does.
Client-facing AI disclosure Per §13 of kerf_ai_disclosure_pattern.md: the artifact is contractor-reviewed and approved, not "AI-assisted by Kerf." The contractor signs · the contractor is accountable. Right Hand built the draft, but the artifact is the contractor's. No Right Hand branding on the document. The client sees their builder's work, not a robot's output.
