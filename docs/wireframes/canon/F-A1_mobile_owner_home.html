F-A1 · Mobile Owner Home · Right Hand launchpad

The new mental model proven in one surface. Owner opens Kerf · sees the urgent decision · sees business state · sees what's queued · sees what just happened on the jobs. Five sections answer the founder's five brain-questions plus the urgent now: One Thing for what's most important, Business state for where guys / money / closing / clients-happy stand right now, On Deck for what's queued, The Pulse for what just happened. Bar contract: 3 mandatory anchors (Home · Speak · More) + 2 user-customizable slots (Projects + Clients as Owner defaults). Built against Operating Model Canon v1 + IA v3.

Frame · F-A1 · Mobile Owner Home · default state (no expansion)

Layer 1 of the disclosure pattern — what the operator sees without doing anything. Every card glance-summarizes in ~10–15 words; deeper context one tap away (Layer 2 expand or Layer 3 drill-down). Plain-English throughout — no AltitudePacket / Policy Gate / cohort vocabulary leaks into operator copy. Pulse items follow event · object · consequence grammar; every item ends in a business consequence the operator can act on.

Sections (top to bottom)

  • One Thing — single most urgent decision; one tap to Decision Card (F-B1)
  • Business state — 4-cell strip answering the brain-questions canon
  • On Deck · 3 — queued items, plain-English with severity dots
  • The Pulse · today — event · object · consequence stream, chronological

Bottom bar contract

  • MPos 1 · Home (mandatory anchor)
  • SPos 2 · Projects (Owner default; swappable)
  • MPos 3 · 🎤 Speak (mandatory · center FAB)
  • SPos 4 · Clients (Owner default; swappable)
  • MPos 5 · More (mandatory · sidebar opener)

The 5 brain-questions

  • Where are my guys at? → cell 1
  • Where is my money at? → cell 2
  • What's closing or drifting? → cell 3 (captures wrap-up + slippage in one scan)
  • Client health → cell 4 (operational scan; covers happy + at-risk)
  • What opportunities are out there? → surfaces via On Deck items (e.g., "Avenida Florencia quiet 3 days") + Pulse

Layered disclosure

  • Layer 1 · Glance — what you see (this state)
  • Layer 2 · Tap-for-more — tap "trust check passed" → expand inline with the four checks; tap a brain-cell → slide-over with the underlying breakdown
  • Layer 3 · Full detail — One Thing CTA → Decision Card (F-B1); Pulse item → relevant project drill

AI disclosure (§13)

  • "AI-assisted by Kerf Right Hand · Review before approval" on the One Thing card (operator surface convention)
  • No "Powered by Llama" or provider attribution anywhere in product UI

Plain-English discipline

  • "$14,820 · 5 scope lines · trust check passed" not "AltitudePacket · 4/4 Policy Gate cleared"
  • "Draft proposal ready" not "system_final_amount_cents emitted"
  • Canon vocabulary lives in audit substrate (Settings → Audit), not operator-rendered body

Pulse grammar

  • Every item: event · object · consequence
  • No event without a consequence — that would be activity exhaust
  • Consequences color-coded: green = forward motion, amber = needs attention, violet = informational

Routes (downstream)

  • One Thing CTA → /decisions/[id] (F-B1)
  • Pulse · field walk captured → /projects/[id]/field
  • Pulse · drift flagged → /projects/[id]/schedule
  • Pulse · invoice overdue → /money/ar/[invoice]
  • Brain cell · Money → /money/ar
  • "See queue →" → /home/on-deck (filterable)
  • "Full stream →" → /home/pulse (time-windowed)
  • 🎤 Speak → context-bound capture session
  • More → sidebar overlay (F-D1)

Forbidden on this surface

  • No "Create New" hub or tool grid
  • No equal-weight tile dashboard
  • No agent name surfaced as a feature ("Drift agent flagged…")
  • No AltitudePacket / DecisionPacket / Policy Gate / cohort / archetype / V1-V18 in operator copy
  • No notifications inbox separate from On Deck + Pulse

Doctrine bound

  • canonKerf is not a launcher · Home leads with continuity, not tools
  • canonRight Hand is the continuity layer · every section speaks through it
  • canonOutputs are artifacts · the One Thing produces a Decision Card
  • canonParallel agent work stays invisible · Pulse surfaces consequences, not agents
  • canonSimplicity beats exhaustiveness · 4 sections · 3 mandatory + 2 swap slots
