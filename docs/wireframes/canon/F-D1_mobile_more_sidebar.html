What this frame proves

One primitive · multiple projections. The 5-slot bar contract + More sidebar overlay is structurally identical across both frames. What changes is the projection through Layer B (role) and Layer C (locale) — exactly as the Operating Model Canon's three-layer separation specifies.

  • Layer A · same: 9 business graph domains, More slot opens overflow, sidebar structure identical (domains · System · Tier-C hook).
  • Layer B · different: Owner sees all 6 non-bar domains + Audit + Tier-C hook. Field Hand sees role-filtered 4 items + Settings only. Same primitive, role-filtered content.
  • Layer C · different: EN tenant shows English labels. ES tenant shows Spanish labels. Right Hand ↔ Mano Derecha. Speak ↔ Habla. Settings ↔ Configuración. Same agent identity carried in both languages as constant cue.

Why this matters: Spanish-native is not a Field-only feature. Owners running tenants in San Diego or Miami with Spanish-speaking admin teams get the full UI in Spanish. PM and Admin/Ops roots get the same treatment when their tier roots wireframe. The platform is bilingual by design at the agent layer, not bilingual-as-translation-afterthought.

Per kerf_agent_naming.md: "Tenant-level locale picks the UI language; the agent identity carries both languages on the primary voice button as a constant cue." Ships V2.1 Paid Beta. Architecture Phase 1 commitment, not V2 polish.
