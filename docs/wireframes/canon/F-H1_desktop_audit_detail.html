What this surface is

The substrate destination. Reached via "View audit detail" deep-link from F-B1/F-B2 Decision Card. Per feedback_audit_deep_link_not_top_nav.md, audit lives one click deep at Settings → Audit, never top nav. Anyone navigating here is explicitly asking for trust detail.

Canon vocabulary IS allowed here

  • This is the one operator surface where canon vocabulary surfaces directly — AltitudePacket / DecisionPacket / validator IDs / cohort / archetype / variance band / P25-P75 / data_class / operator-private
  • Plain-English rule still applies in operator-facing primary surfaces — this surface is the inverse: the substrate destination shows the receipts
  • Per feedback_translate_canon_vocabulary_at_surface.md: "Audit substrate (Settings → Audit) may surface canon vocabulary for trust review"

Five sections

  • Validator wall · 9 checks — V1/V2/V6/V7/V8/V9/V12/V17/V18 canonical validators. Each card has validator ID, name, detail. All passed for an approved decision. A failed validator would render in red and block approval at F-G1.
  • Source references · 4 — same 4 plain-English sources from F-B1/B2 Decision Card, now annotated with their canon anchors: transcript hash · cohort ID + window + variance band · RoomPlan model ID + use-label · clarification event IDs
  • Pricing breakdown — model-suggested vs operator-final per scope line, with delta. Subtotal row highlighted. Storage-as-cents shown on the DecisionPacket card; display-as-dollars on the pricing table.
  • Approval chain — vertical timeline from capture → transcript → draft → send-gate resolution → decision approval → (pending external send). Each event has actor, timestamp, anchor reference.
  • Role visibility — Owner / PM / Admin / Field / Sub / Client rows with data_class and permission level. Permission lattice visible. Operator-private content surfaces here for the Owner but not for any other role.

Right rail · DecisionPacket card

  • Packet ID prominently displayed (altpkt_co-002_a7f3) — copyable for support / dispute / audit export
  • All packet fields shown: type · project · modifies · final amount (cents + dollar display) · approver · timestamp · model version · validator status · parent packets
  • Below: role visibility list with data_class per role
  • Below: vendor protection reminder — privileged-class never transits external LLM, tenant-keyed encryption for operator-private fields

Doctrine bound

  • canonAudit at Settings · one click deep · never top nav
  • canonCanon vocabulary surfaces here (validator IDs, packet IDs, cohort/archetype/variance) — the substrate destination is where trust receipts live
  • canonPermission lattice visible — operator sees what each role sees, including their own redaction class
  • canonMoney displayed dual: cents (storage truth) on the packet card + dollars (operator display) on the pricing table
  • canonVendor protection reminder visible — privileged-class isolation + tenant-keyed encryption · the public-trust posture
  • canonDispute path exists · "Dispute this decision" opens a structured grievance flow (operator-initiated reopen)

What this surface enables

  • Trust verification · operator can confirm any specific aspect of the decision (which source · which validator · which model version)
  • Audit export · PDF export of the full receipt for legal-defensibility (De Wall pattern)
  • Reproducibility · packet ID + model version + cohort window + source IDs let any future analyst reproduce the decision logic
  • Dispute initiation · operator can open a structured dispute that reopens the decision lifecycle
  • Per-project audit lens · same substrate surfaces via Projects > [project] > Audit sub-tab filtered to that project's decisions

Routes

  • Source reference "Open →" → respective source destination (transcript · cohort comparable · RoomPlan model · clarification event)
  • Validator "Validator definitions →" → canon definition page for V1-V18 validators
  • Pricing "Export CSV →" → CSV export of the pricing breakdown
  • "Export receipt (PDF)" → full audit receipt as PDF
  • "Dispute this decision" → structured dispute flow · creates a dispute packet · reopens decision lifecycle
  • "Back to Decision Card" → F-B1 (mobile) or F-B2 (desktop) Decision Card
  • Breadcrumb "Settings · Audit · Decisions" → respective audit destination scope

What this surface guarantees Every decision has a receipt. No "the AI did it" hand-wave. The validator wall, source references, pricing breakdown, approval chain, role visibility, and DecisionPacket metadata are all reproducible. If the decision is ever disputed in legal proceedings or audit, the full trail is one click deep — at the substrate destination, not the operator surface.
What this surface refuses to be Not a chronological event log (that's a separate destination). Not an operator-primary surface (operator surfaces stay plain English). Not a Kerf-staff backdoor (vendor protection + tenant-keyed encryption hold). F-H1 is the trust receipt for one decision, the per-decision substrate slice of the Audit destination.
