4-question rule

  • Macro / comparison / detail? Detail — single-operator account editor (not a settings landing)
  • List / grid / both? Neither — form sections · Grid view: NO
  • Cognitive job? Edit username · credentials · contact info · avatar · notification channels · UI language
  • Hidden until drill? Password hash · full audit history · permissions matrix (F-SP1 Permissions card)

Memory exposure

  • Identity-layer editor · editable by consequence · §13 audit on every save · password is write-only substrate — never in model or Right Hand working memory · profile fields update operator identity only

Fields (Lane 7)

  • Username · editable · uniqueness check within tenant
  • Password · change with current-password confirm
  • Personal info · full name · email · phone · timezone
  • Avatar · upload + preview · initials fallback
  • Notifications · email · push · SMS · Slack/Teams toggles
  • Language · English / Español · ties to Spanish-native canon
  • Save / Cancel · audit trail on any change

  • "Right Hand, update my phone number to …"
  • "Right Hand, switch my language to Spanish"

  • F-SP1 Account card → this surface
  • Cancel → F-SP1
