F-G1r · Spec

4-question rule

  • Macro / comparison / detail? Detail.
  • List / grid / both? Neither. Two-column send-gate layout only.
  • Operator cognitive job? Decide whether this draft is ready to send and understand where each major piece came from.
  • What stays hidden until drill-down? Transcript internals, audit trail, and older draft details beyond lightweight version history.

Why multi-panel is allowed here

  • Send is high-commitment and client-facing.
  • Provenance helps the operator trust the outgoing artifact.
  • The right panel stays disciplined: no extra dashboards, only source explanation.

Pattern lock

  • Left = client-facing draft.
  • Right = provenance and lightweight history.
  • Transcript review remains a deeper repair surface, not part of the main page furniture.
What this enforcesDraft Review is the send gate. It lets the operator see the artifact as the client will see it and check the provenance of each important section before approving.
What this refuses to beNot an audit console, not a transcript workspace, and not a pricing spreadsheet. It stays centered on one decision: ready to send or not.
