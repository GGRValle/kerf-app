F-MN6a Grid · Spec

4-question rule

  • Macro / comparison / detail? Comparison (delta-magnitude posture by aging × amount)
  • List / grid / both? Grid (sibling toggle to Frame 1 List)
  • Cognitive job? Which biggest oldest invoices need attention first?
  • Hidden until drill? Invoice line items · payment history (drill via tile click)

Tile anatomy

  • 2-column mobile grid · 17 tiles · same 17 invoices as List frame
  • gt-name = client · gt-phase = aging bucket · gt-chip = state chip · gt-rhythm = open $
  • Red chip = call needed (90+ Escalated · 61-90 At-risk)
  • Amber chip + amber left-border on tile = past due (31-60d)
  • Neutral chip = normal flow (0-30d)
  • `.grid-tile.risk` class not used — Lane 3 doctrine: red is chip-tier only
What this surface enforcesAR as an action queue by age. Oldest first; call/reminder actions appear only on at-risk rows. Section totals + RH summary surface the macro reality (DSO posture, escalation count).
What this surface refuses to beNot the general ledger. Not invoice composition (drill to project for line items). F-MN6a compares open receivables for action — drill to client Money sub-tab for full payment history.
