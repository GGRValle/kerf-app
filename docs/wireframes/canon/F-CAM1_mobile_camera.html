F-CAM1 · Spec

Routing gate

  • A project or lead must be selected before the camera opens (your note) — captures always file to a job, never orphan.
  • Field worker clocked in → current job pre-selected (one tap). Manager → search / recents / new lead.
  • The selected job rides as a chip in the camera chrome; everything saves there.

Where each mode files

  • Photo → job photos / today's Daily Log.
  • Walkthru → Daily Log video note + transcript.
  • Scan → job Documents (plans / site info) — and is offered to Right Hand for estimates / change orders.

One camera, everywhere

  • This is the Right Hand camera. Anywhere the app needs a camera — "add a photo" inside a job, a Daily Log, an estimate — this interface opens.
  • Launched with job context → the routing gate is skipped (job already known) and Done returns you to where you were.
  • Launched from the bottom-bar Camera (no context) → the routing gate (Frame 1) comes first.

Consistent exit

  • Done is on all three modes (gold pill, same spot) — one way out, save everything to the job, return to origin. Per-mode shutter captures; Done finishes.
Frames 2–4 · One camera · three modes (swipe Walkthru · Photo · Scan)

⚠ Layout = a camera, not a form (build proportions exactly this way). The phones above show the intended scale. Critical, because the first build rendered as a centered web form:
• Viewfinder is full-bleed. The live preview is the background — it fills the entire frame edge-to-edge, top chrome and bottom controls float over it. No centered circle in a padded panel, no margins around the preview. (Empty/no-permission state still fills the frame — a dark preview with a small centered hint, not a boxed placeholder.)
• Controls are a thin overlay at the bottom, not stacked rows with even spacing. Zoom pills → mode strip → shutter row sit as one compact cluster over the preview, ~bottom third max.
• The shutter is the dominant control — the largest element, centered. Done is a small text/pill affordance to its right; the left slot is the last-capture thumbnail. Room scan is NOT on the capture row — it's a separate entry (own flow, F-RC1), so the capture row reads as one big shutter, not three equal buttons.
• Top chrome (settings · flash · flip · close) + job chip overlay the top of the preview; they don't push it down.

  • Full-bleed viewfinder — preview fills the frame; chrome floats over it. Not a centered circle in a box.
  • Shutter dominant + centered; Done = small affordance right; thumbnail left. Room scan off the capture row.

The default

  • Plain shutter; tap to capture. Files to the job's photos / today's Daily Log.
  • Top chrome (your sketch): Settings (left), Flash + front/rear flip (right), job chip center, close — overlaid on the preview top.
  • Zoom pills (.5 / 1× / 3×) above the mode switch.

Walkthru

Brings in Right Hand

  • Center mode = video + live mic. The gold "Right Hand listening · REC" pill + waveform show you're speaking and recording (your "add mic for consistency").
  • You narrate as you film; live caption confirms it's hearing you.
  • Done saves the video + transcript to the job's Daily Log and routes it to Right Hand for synthesis (notes → draft).

Scan

Plans & site docs → job document

  • Document capture (edge-snap, multi-page) — not LiDAR. For scanning plans, spec sheets, on-site info.
  • Saves to the job's Documents and is offered to Right Hand as source for estimates / change orders (feeds F-CHG1 / F-EST1 Add-item + scope).
  • LiDAR room scan stays separate (F-RC1).
